# Test-Driven Development (TDD) Workflow
**ID**: `M10-L02`  
**Type**: lesson  
**Duration**: 30-60 menit  
**Tags**: python, tutorial

## Tujuan Pembelajaran
Follow Red-Green-Refactor cycle, write tests before implementation.

## Materi

### Konsep Dasar
**TDD**: Development process di mana kita menulis test terlebih dahulu sebelum implementasi kode. Cycle:
1. **Red**: Tulis test yang gagal (belum ada implementasi).
2. **Green**: Tulis minimal kode agar test berhasil (pass).
3. **Refactor**: Perbaiki kode tanpa mengubah behavior (pastikan test tetap pass).

**Benefits**:
- Desain API dari sudut pandang pengguna
- Confidence dalam refactoring
- Documentation via tests
- Prevent bugs

### TDD Steps Example
```python
# test_calculator.py
def test_add():
    assert add(2, 3) == 5

# calculator.py
def add(a, b):
    return a + b  # implementasi paling sederhana
```

**Red**: test gagal karena `add` belum ada.
**Green**: implement `add` paling sederhana.
**Refactor**: improve implementation (misal: handle edge cases) sambil menjaga test tetap pass.

### Practical Tips
- Start dengan test-case yang jelas
- Test satu behavior per test function
- Use descriptive test names: `test_function_should_...`
- Keep tests fast dan independent
- Use fixtures untuk setup common data
- Run tests frequently

### Detail Teknis
- TDD bukan berarti test 100% coverage; fokus pada behavior penting
- Bisa test exception: `with pytest.raises(ValueError): func(...)`
- Refactor bisa mencakup: rename variables, extract functions, improve algorithms
- Tetap maintainable: test code juga perlu dibersihkan secara berkala
- TDD cocok untuk功能 development; untuk exploration/prototyping mungkin kurang

## Latihan

### Soal 1
Gunakan TDD untuk mengembangkan fungsi `factorial(n)` yang menghitung factorial dari non-negative integer. Tulis test terlebih dahulu (dengan pytest). Lalu implementasi paling sederhana agar test pass. Kemudian refactor untuk handle edge cases (factorial(0)=1, negative raise ValueError).

```python
# test_factorial_tdd.py
import pytest

def factorial(n):
    # Implementation awal (hanya untuk passing test)
    pass

@pytest.mark.parametrize("n,expected", [
    (0, 1),
    (1, 1),
    (5, 120),
])
def test_factorial(n, expected):
    assert factorial(n) == expected

def test_factorial_negative():
    with pytest.raises(ValueError):
        factorial(-1)
```

**Expected Output** (saat `pytest`):
```
3 passed in 0.01s
```

### Soal 2
TDD untuk class `BankAccount`:
- `__init__(self, initial_balance=0)` set balance
- `deposit(amount)` tambah balance, return None; amount harus >0, else ValueError
- `withdraw(amount)` kurangi balance, return None; amount >0 dan <= balance, else ValueError
- `get_balance()` return balance

Tulis test terlebih dahulu (menggunakan pytest), lalu implement, lalu refactor.

```python
# test_bank_account_tdd.py
class BankAccount:
    # Tulis implementasi
    pass

def test_initial_balance():
    acc = BankAccount()
    assert acc.get_balance() == 0

def test_deposit():
    acc = BankAccount()
    acc.deposit(100)
    assert acc.get_balance() == 100

def test_deposit_negative():
    acc = BankAccount()
    with pytest.raises(ValueError):
        acc.deposit(-50)

def test_withdraw():
    acc = BankAccount(100)
    acc.withdraw(30)
    assert acc.get_balance() == 70

def test_withdraw_insufficient():
    acc = BankAccount(50)
    with pytest.raises(ValueError):
        acc.withdraw(100)
```

**Expected Output**:
```
5 passed in 0.02s
```
