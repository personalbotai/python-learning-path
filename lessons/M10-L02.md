## Materi

### Konsep Dasar

**Test-Driven Development (TDD)** adalah development process di mana kita menulis test **sebelum** menulis implementation code. Cycle:

1. **Red**: Write failing test (must fail because feature doesn't exist)
2. **Green**: Write minimal code to make test pass
3. **Refactor**: Improve code while keeping tests green

**Benefits:**
- Better design (think about API before implementation)
- Confidence to refactor (tests catch regressions)
- Documentation (tests show how code should be used)
- Fewer bugs (test coverage from start)

### Detail Teknis

**1. TDD Cycle Example**

```python
# Feature: String reversal

# Step 1: RED - Write test first (test_reverse.py)
def test_reverse_string():
    assert reverse_string("hello") == "olleh"
    assert reverse_string("") == ""
    assert reverse_string("a") == "a"

# Step 2: GREEN - Minimal code to pass
def reverse_string(s):
    return s[::-1]  # simplest implementation

# Step 3: REFACTOR - Improve if needed (maybe add edge case handling)
def reverse_string(s):
    # Could add type hints, validation
    if not isinstance(s, str):
        raise TypeError("Expected string")
    return s[::-1]
```

**2. Writing Good Tests**

```python
# BAD: vague test
def test_add():
    assert add(2, 3) == 5

# GOOD: descriptive name + clear assertions
def test_add_positive_numbers():
    """Addition of two positive integers should return correct sum"""
    assert add(2, 3) == 5
    assert add(10, 20) == 30

def test_add_with_zero():
    assert add(0, 5) == 5
    assert add(5, 0) == 5

def test_add_negative_numbers():
    assert add(-1, -1) == -2
    assert add(-5, 3) == -2
```

**3. TDD for Edge Cases**

```python
# Feature: Safe division

# First test (happy path)
def test_divide_positive():
    assert divide(10, 2) == 5

# Second test (edge case: zero divisor)
def test_divide_by_zero():
    with pytest.raises(ValueError):
        divide(10, 0)

# Third test (edge case: negative numbers)
def test_divide_negative():
    assert divide(-10, 2) == -5
    assert divide(10, -2) == -5

# Implementation:
def divide(a, b):
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b
```

**4. Using Fixtures in TDD**

```python
@pytest.fixture
def empty_stack():
    """Provide fresh stack for each test"""
    return []

def test_stack_push(empty_stack):
    empty_stack.append(1)
    assert len(empty_stack) == 1

def test_stack_pop(empty_stack):
    empty_stack.append(1)
    value = empty_stack.pop()
    assert value == 1
    assert len(empty_stack) == 0
```

**5. TDD for OOP**

```python
# Test first, then class
class TestBankAccount:
    def test_initial_balance(self):
        acc = BankAccount("123", "Alice")
        assert acc.balance == 0
    
    def test_deposit(self):
        acc = BankAccount("123", "Alice")
        acc.deposit(100)
        assert acc.balance == 100
    
    def test_withdraw_insufficient(self):
        acc = BankAccount("123", "Alice", 50)
        with pytest.raises(ValueError):
            acc.withdraw(100)
```

**6. Red-Green-Refactor in Practice**

```python
# RED: Write test that fails
def test_calculate_tax():
    """Tax should be 10% of income"""
    assert calculate_tax(1000) == 100

# GREEN: Minimal implementation
def calculate_tax(income):
    return income * 0.1

# RED: Add more test (edge case)
def test_calculate_tax_zero():
    assert calculate_tax(0) == 0

# GREEN: Already passes! No code change needed.

# RED: Add test for negative income (should raise error)
def test_calculate_tax_negative():
    with pytest.raises(ValueError):
        calculate_tax(-100)

# GREEN: Update implementation
def calculate_tax(income):
    if income < 0:
        raise ValueError("Income cannot be negative")
    return income * 0.1

# REFACTOR: Maybe add type hints, docstring
def calculate_tax(income: float) -> float:
    """
    Calculate tax at 10% rate.
    
    Args:
        income: Positive number
    
    Returns:
        Tax amount
    
    Raises:
        ValueError: If income is negative
    """
    if income < 0:
        raise ValueError("Income cannot be negative")
    return income * 0.1
```

**7. TDD Benefits & Pitfalls**

```python
# Good TDD:
# - Tests are fast, isolated, repeatable
# - Tests fail for right reason (missing feature)
# - Tests help design API (think about usage first)

# Bad TDD:
# - Writing implementation first anyway
# - Tests that always pass (no assertions)
# - Testing implementation details (brittle)
# - Slow tests (I/O in unit tests)
```

### Soal 2: TDD for Shopping Cart

Feature: Shopping cart dengan methods `add_item`, `remove_item`, `total_price`. Use TDD:

```python
# test_cart.py
class TestShoppingCart:
    def test_initial_empty(self):
        cart = ShoppingCart()
        assert cart.items == {}
        assert cart.total() == 0
    
    def test_add_item(self):
        cart = ShoppingCart()
        cart.add_item("apple", 1.0, 2)
        assert cart.items == {"apple": {"price": 1.0, "qty": 2}}
        assert cart.total() == 2.0
    
    def test_add_same_item_increments_qty(self):
        cart = ShoppingCart()
        cart.add_item("apple", 1.0, 2)
        cart.add_item("apple", 1.0, 3)
        assert cart.items["apple"]["qty"] == 5
        assert cart.total() == 5.0
    
    def test_remove_item(self):
        cart = ShoppingCart()
        cart.add_item("apple", 1.0, 2)
        cart.remove_item("apple")
        assert "apple" not in cart.items
    
    def test_remove_item_partial(self):
        cart = ShoppingCart()
        cart.add_item("apple", 1.0, 5)
        cart.remove_item("apple", qty=2)
        assert cart.items["apple"]["qty"] == 3
    
    def test_remove_nonexistent_raises(self):
        cart = ShoppingCart()
        with pytest.raises(KeyError):
            cart.remove_item("banana")
```

Implement `ShoppingCart` class yang membuat semua test di atas pass.

### Soal 3: TDD for Password Validator

Feature: Password validator dengan rules:
- Minimal 8 characters
- Minimal 1 uppercase
- Minimal 1 lowercase
- Minimal 1 digit
- Minimal 1 special character (!@#$%^&*)

```python
# test_password.py
import pytest
from password import validate_password

def test_valid_password():
    assert validate_password("StrongP@ss123") == True

def test_too_short():
    assert validate_password("Short1!") == False

def test_missing_uppercase():
    assert validate_password("lowercase123!") == False

def test_missing_lowercase():
    assert validate_password("UPPERCASE123!") == False

def test_missing_digit():
    assert validate_password("NoDigitHere!") == False

def test_missing_special():
    assert validate_password("MissingSpecial123") == False

def test_empty_password():
    assert validate_password("") == False

def test_multiple_errors():
    # Should return False, not raise exception
    assert validate_password("weak") == False
```

Implement `validate_password(password: str) -> bool` dengan TDD.
