## Materi

### Konsep Dasar

**Inheritance** ("is-a"): Child inherits from Parent. Tight coupling, fragile base class problem.

**Composition** ("has-a" dengan ownership): Class contains other objects, controlling their lifecycle. Strong relationship.

**Aggregation** ("has-a" tanpa ownership): Class contains references to other objects, but lifecycle independent. Weak relationship.

```
Inheritance:     Car extends Vehicle  (Car IS-A Vehicle)
Composition:     Car has an Engine    (Car OWNS Engine, Engine destroyed with Car)
Aggregation:     Car uses a Driver    (Car HAS Driver, Driver exists independently)
```

### Detail Teknis

**1. Composition (Ownership)**

```python
class Engine:
    def __init__(self, horsepower):
        self.horsepower = horsepower
    
    def start(self):
        return f"Engine with {self.horsepower} HP started"

class Car:
    def __init__(self, make, model, engine):
        self.make = make
        self.model = model
        self.engine = engine  # Composition: Car owns Engine
    
    def drive(self):
        return f"{self.make} {self.model}: {self.engine.start()}"

# Usage
engine = Engine(150)
car = Car("Toyota", "Camry", engine)
print(car.drive())  # Toyota Camry: Engine with 150 HP started
# Engine lifecycle tied to Car (if Car deleted, Engine might be deleted too)
```

**2. Aggregation (Reference without Ownership)**

```python
class Driver:
    def __init__(self, name):
        self.name = name
    
    def drive(self):
        return f"{self.name} is driving"

class Car:
    def __init__(self, make, model):
        self.make = make
        self.model = model
        self.driver = None  # Aggregation: Car uses Driver, but doesn't own
    
    def assign_driver(self, driver):
        self.driver = driver
    
    def drive(self):
        if self.driver:
            return f"{self.make} {self.model}: {self.driver.drive()}"
        return f"{self.make} {self.model}: No driver assigned!"

# Usage
driver = Driver("Alice")
car = Car("Honda", "Civic")
car.assign_driver(driver)
print(car.drive())  # Honda Civic: Alice is driving

# Driver can exist independently and be shared
car2 = Car("Tesla", "Model 3")
car2.assign_driver(driver)  # Same driver instance
print(car2.drive())  # Tesla Model 3: Alice is driving
# Deleting car doesn't delete driver
```

**3. Comparison Table**

| Aspect | Inheritance | Composition | Aggregation |
|--------|-------------|-------------|-------------|
| Relationship | IS-A | HAS-A (owns) | HAS-A (uses) |
| Coupling | Tight | Loose | Loose |
| Lifecycle | Child inherits parent | Child manages parent's lifecycle | Independent |
| Reuse | Through inheritance | Through object containment | Through references |
| Flexibility | Low (single inheritance in Python) | High (can swap components) | High (can share objects) |
| Changes | Affect all subclasses | Isolated to containing class | Isolated |

**4. Favor Composition Over Inheritance**

```python
# INHERITANCE (problematic)
class Email:
    def send(self, message):
        print(f"Sending email: {message}")

class SMS(Email):  # SMS IS-A Email? Not really!
    def send(self, message):
        print(f"Sending SMS: {message}")

# COMPOSITION (better)
class MessageSender:
    def send(self, message):
        raise NotImplementedError

class EmailSender(MessageSender):
    def send(self, message):
        print(f"Sending email: {message}")

class SMSSender(MessageSender):
    def send(self, message):
        print(f"Sending SMS: {message}")

class Notification:
    def __init__(self, sender: MessageSender):
        self.sender = sender  # Composition
    
    def notify(self, message):
        self.sender.send(message)

# Usage
email_notif = Notification(EmailSender())
sms_notif = Notification(SMSSender())
email_notif.notify("Hello")  # Sending email: Hello
sms_notif.notify("Hello")   # Sending SMS: Hello
# Easy to add new sender types without modifying Notification
```

**5. Dependency Injection (DI)**

```python
# Without DI (tight coupling)
class Order:
    def __init__(self):
        self.email_service = EmailService()  # Hard-coded dependency
    
    def send_confirmation(self):
        self.email_service.send(...)

# With DI (loose coupling)
class Order:
    def __init__(self, email_service):  # Inject dependency
        self.email_service = email_service
    
    def send_confirmation(self):
        self.email_service.send(...)

# Usage
order = Order(SMSService())  # Can swap implementations
```

**6. Mixing Composition & Inheritance**

```python
# Inheritance for IS-A relationship
class Animal:
    def move(self):
        pass

class Bird(Animal):  # Bird IS-A Animal
    def __init__(self, wings):
        self.wings = wings  # Composition: Bird HAS Wings
    
    def move(self):
        return f"Flying with {self.wings} wings"

class Fish(Animal):
    def move(self):
        return "Swimming"

# Polymorphic behavior + composition
animals = [Bird(2), Fish()]
for a in animals:
    print(a.move())
```

**7. When to Use Which**

```
Use INHERITANCE when:
- There is a clear "is-a" relationship
- You need polymorphism (shared interface)
- Parent class is stable and designed for extension
- You want to override/extend behavior

Use COMPOSITION when:
- You need to combine multiple behaviors (multiple inheritance alternative)
- You want to change behavior at runtime (swap components)
- You want to avoid fragile base class problem
- You need to reuse code across unrelated classes

Use AGGREGATION when:
- Objects have independent lifecycles
- Relationship is "uses" not "owns"
- Same object shared by multiple containers
```

**8. Design Patterns that Use Composition**

```python
# Strategy Pattern (composition of algorithms)
class SortStrategy:
    def sort(self, data):
        pass

class QuickSort(SortStrategy):
    def sort(self, data):
        return sorted(data)  # simplified

class BubbleSort(SortStrategy):
    def sort(self, data):
        # bubble sort implementation
        return data

class Sorter:
    def __init__(self, strategy: SortStrategy):
        self.strategy = strategy
    
    def set_strategy(self, strategy):
        self.strategy = strategy
    
    def sort(self, data):
        return self.strategy.sort(data)

# Can swap algorithms at runtime
sorter = Sorter(QuickSort())
sorter.sort([3,1,2])
sorter.set_strategy(BubbleSort())
sorter.sort([3,1,2])

# Observer Pattern (aggregation of listeners)
class Event:
    def __init__(self):
        self.listeners = []  # Aggregation: Event has many listeners
    
    def add_listener(self, listener):
        self.listeners.append(listener)
    
    def fire(self, data):
        for listener in self.listeners:
            listener(data)
```

### Soal 1: E-commerce System dengan Composition

Buat sistem e-commerce sederhana dengan composition:

```python
class PaymentProcessor:
    def process(self, amount):
        raise NotImplementedError

class CreditCardProcessor(PaymentProcessor):
    def __init__(self, card_number):
        self.card_number = card_number
    
    def process(self, amount):
        return f"Paid ${amount} with CC {self.card_number[-4:]}"

class PayPalProcessor(PaymentProcessor):
    def __init__(self, email):
        self.email = email
    
    def process(self, amount):
        return f"Paid ${amount} via PayPal ({self.email})"

class Order:
    def __init__(self, order_id, payment_processor: PaymentProcessor):
        self.order_id = order_id
        self.payment_processor = payment_processor  # Composition
        self.items = []
        self.total = 0
    
    def add_item(self, name, price, quantity=1):
        self.items.append({"name": name, "price": price, "qty": quantity})
        self.total += price * quantity
    
    def checkout(self):
        result = self.payment_processor.process(self.total)
        return f"Order {self.order_id}: {result}"

def main():
    # Order with credit card
    cc_processor = CreditCardProcessor("1234-5678-9012-3456")
    order1 = Order("ORD001", cc_processor)
    order1.add_item("Laptop", 1000, 1)
    order1.add_item("Mouse", 25, 2)
    print(order1.checkout())  # Paid $1050 with CC 3456
    
    # Order with PayPal
    pp_processor = PayPalProcessor("user@example.com")
    order2 = Order("ORD002", pp_processor)
    order2.add_item("Keyboard", 50, 1)
    print(order2.checkout())
```

### Soal 2: Game Character System

Buat game character system dengan composition untuk abilities:

```python
class Ability:
    def use(self):
        raise NotImplementedError

class Fireball(Ability):
    def use(self):
        return "Casting Fireball! 30 damage"

class Heal(Ability):
    def use(self):
        return "Casting Heal! +50 HP"

class Invisibility(Ability):
    def use(self):
        return "Become invisible for 10 seconds"

class Character:
    def __init__(self, name, health=100):
        self.name = name
        self.health = health
        self.abilities = []  # Composition: Character HAS abilities
    
    def add_ability(self, ability: Ability):
        self.abilities.append(ability)
    
    def use_ability(self, index):
        if 0 <= index < len(self.abilities):
            return self.abilities[index].use()
        return "No such ability"
    
    def __str__(self):
        return f"{self.name} (HP: {self.health})"

def main():
    mage = Character("Gandalf", 80)
    mage.add_ability(Fireball())
    mage.add_ability(Heal())
    mage.add_ability(Invisibility())
    
    print(mage)
    for i in range(3):
        print(f"Ability {i+1}: {mage.use_ability(i)}")
    
    # Characters can share abilities (aggregation)
    warrior = Character("Conan", 150)
    warrior.add_ability(Heal())  # Same Heal instance? Or new?
    # Both can use same ability class but different instances
```

### Soal 3: Refactor Inheritance to Composition

Berikut adalah kode yang berlebihan menggunakan inheritance. Refactor untuk menggunakan composition:

```python
# BEFORE (inheritance hell)
class EmailNotification:
    def send(self, message):
        print(f"Email: {message}")

class SMSNotification:
    def send(self, message):
        print(f"SMS: {message}")

class PushNotification:
    def send(self, message):
        print(f"Push: {message}")

class EmailAlert(EmailNotification):
    def send_alert(self, level):
        self.send(f"Alert level {level}!")

class SMSAlert(SMSNotification):
    def send_alert(self, level):
        self.send(f"Alert level {level}!")

class PushAlert(PushNotification):
    def send_alert(self, level):
        self.send(f"Alert level {level}!")

# AFTER: Refactor dengan composition
class Notifier:
    """Generic notifier that can use any channel"""
    def __init__(self, channel):
        self.channel = channel  # Composition
    
    def send_alert(self, level):
        message = f"Alert level {level}!"
        return self.channel.send(message)

# Usage
email_channel = EmailNotification()
sms_channel = SMSNotification()

email_alert = Notifier(email_channel)
sms_alert = Notifier(sms_channel)

email_alert.send_alert("HIGH")  # Email: Alert level HIGH!
sms_alert.send_alert("LOW")    # SMS: Alert level LOW!
```
