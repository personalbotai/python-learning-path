## Materi

### Konsep Dasar
Operasi file system menggunakan modul `os` dan `shutil`.

**os module**:
```python
import os

os.listdir(path='.')          # list direktori (names only)
os.scandir(path='.')          # iterator of DirEntry (lebih efisien, punya atribut)
os.getcwd()
os.chdir(path)
os.mkdir(path)               # buat satu level
os.makedirs(path, exist_ok=True)  # buat recursively
os.rmdir(path)               # hapus direktori kosong
os.remove(path)              # hapus file
os.removedirs(path)          # hapus recursively (jika kosong)
os.rename(src, dst)
os.replace(src, dst)         # atomic rename
os.scandir() untuk atribut: entry.name, entry.path, entry.is_file(), entry.is_dir()
```

**shutil module** (high-level operations):
```python
import shutil

shutil.copy(src, dst)            # copy file
shutil.copy2(src, dst)           # copy dengan metadata
shutil.copytree(src, dst)        # copy directory tree
shutil.move(src, dst)            # move/rename
shutil.rmtree(path)              # hapus directory tree (berbahaya!)
shutil.disk_usage(path)          # (total, used, free) dalam bytes
```

**os.path** (tambahan):
```python
os.path.exists(path)
os.path.isfile(path)
os.path.isdir(path)
os.path.getsize(path)
```

### Detail Teknis
- `os.scandir()` lebih cepat daripada `os.listdir()` karena mengembalikan `DirEntry` dengan cached type info
- `os.makedirs(..., exist_ok=True)` untuk avoid error jika directory sudah ada
- `shutil.rmtree()` tidak bisa di-undo - hati-hati
- `os.remove()` untuk file, `os.rmdir()` untuk empty directory
- `os.rename()` bisa cross-filesystem? Tidak selalu; gunakan `shutil.move()` untuk cross-filesystem
- `os.walk()` untuk recursive directory traversal (generator)
- Permission errors bisa terjadi; tangkap exceptions
