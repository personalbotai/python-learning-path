## Materi

### Konsep Dasar

**List Comprehension** adalah cara concise untuk membuat list dengan menerapkan expression pada setiap item sequence. Syntax: `[expression for item in iterable if condition]`.

**Basic example:**
```python
# Traditional for loop
squares = []
for x in range(10):
    squares.append(x ** 2)

# List comprehension (lebih pendek dan readable)
squares = [x ** 2 for x in range(10)]
```

### Detail Teknis

**1. Basic List Comprehension**

```python
# Without condition
numbers = [1, 2, 3, 4, 5]
doubled = [x * 2 for x in numbers]  # [2, 4, 6, 8, 10]

# With condition (filter)
evens = [x for x in numbers if x % 2 == 0]  # [2, 4]
uppercase = [s.upper() for s in ["hello", "world"] if len(s) > 3]  # ["HELLO", "WORLD"]

# Complex expression
sentences = ["hello world", "python is great"]
words = [word for sentence in sentences for word in sentence.split()]
# ["hello", "world", "python", "is", "great"]
```

**2. Nested List Comprehensions**

```python
# Flatten matrix 2D
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
flattened = [num for row in matrix for num in row]
# [1,2,3,4,5,6,7,8,9]

# With condition
flattened_even = [num for row in matrix for num in row if num % 2 == 0]
# [2,4,6,8]

# Transpose matrix
transposed = [[row[i] for row in matrix] for i in range(len(matrix[0]))]
# [[1,4,7], [2,5,8], [3,6,9]]
```

**3. Dictionary Comprehensions**

```python
# Create dict from iterable
squares_dict = {x: x**2 for x in range(5)}  # {0:0, 1:1, 2:4, 3:9, 4:16}

# Transform existing dict
prices = {"apple": 1.0, "banana": 0.5, "cherry": 2.0}
discounted = {k: v * 0.9 for k, v in prices.items()}  # 10% discount

# Filter dict
expensive = {k: v for k, v in prices.items() if v > 1.0}  # {"cherry": 2.0}

# Swap keys and values (if values are hashable)
swapped = {v: k for k, v in prices.items()}  # {1.0: "apple", 0.5: "banana", ...}
```

**4. Set Comprehensions**

```python
# Unique squares
unique_squares = {x**2 for x in [1, 2, 2, 3, 3, 3]}  # {1, 4, 9}

# Extract unique characters
unique_chars = {c for s in ["hello", "world"] for c in s}  # {'d','e','h','l','o','r','w'}

# Filter unique
words = ["apple", "banana", "apricot", "blueberry"]
starts_with_a = {w for w in words if w.startswith('a')}  # {"apple", "apricot"}
```

**5. Generator Expressions**

```python
# List comprehension creates full list in memory
list_comp = [x**2 for x in range(1000000)]  # uses lots of memory

# Generator expression - lazy evaluation
gen_expr = (x**2 for x in range(1000000))  # minimal memory, computes on demand

# Consume generator
for value in gen_expr:
    if value > 100:
        break

# Convert to list if needed (but defeats lazy purpose)
list_from_gen = list(gen_expr)
```

**6. Comprehensions with `enumerate()` and `zip()`**

```python
# enumerate for index + value
fruits = ["apple", "banana", "cherry"]
indexed = [(i, fruit) for i, fruit in enumerate(fruits)]
# [(0, 'apple'), (1, 'banana'), (2, 'cherry')]

# With start index
indexed = [(i, fruit) for i, fruit in enumerate(fruits, start=1)]
# [(1, 'apple'), (2, 'banana'), (3, 'cherry')]

# zip multiple lists
names = ["Alice", "Bob"]
ages = [25, 30]
combined = [(name, age) for name, age in zip(names, ages)]
# [('Alice', 25), ('Bob', 30)]

# zip with enumerate
indexed_names = [(i, name) for i, (name, age) in enumerate(zip(names, ages))]
```

**7. When to Use vs When Not to Use**

```python
# ✅ GOOD: Simple transformation, readability
result = [x.upper() for x in words if len(x) > 3]

# ❌ BAD: Complex logic, multiple nested loops (use regular for loop)
result = []
for x in matrix:
    for y in x:
        if y > 0:
            for z in y:
                if z % 2 == 0:
                    result.append(complex_calculation(z))
# Better as nested for loops for readability

# ❌ BAD: Side effects (modify external list)
# List comprehension should produce new list, not have side effects
result = []
[result.append(x) for x in data]  # ANTI-PATTERN! Use for loop instead

# ✅ GOOD: Functional style
filtered = list(filter(lambda x: x > 0, data))
mapped = list(map(lambda x: x**2, data))
# But comprehensions often more readable:
filtered = [x for x in data if x > 0]
mapped = [x**2 for x in data]
```

**8. Performance**

```python
import timeit

# List comprehension vs for loop + append
time_comp = timeit.timeit('[x**2 for x in range(1000)]', number=1000)
time_loop = timeit.timeit('''
result = []
for x in range(1000):
    result.append(x**2)
''', number=1000)

print(f"Comprehension: {time_comp:.4f}s")
print(f"For loop: {time_loop:.4f}s")
# Comprehension usually slightly faster (optimized bytecode)

# Generator expression for memory efficiency (but slower if need all)
gen_time = timeit.timeit('list(x**2 for x in range(1000))', number=1000)
# Similar speed but uses less memory for large ranges
```
