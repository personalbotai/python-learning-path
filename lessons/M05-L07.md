## Tujuan Pembelajaran
Understand polymorphism in Python, embrace duck typing, and use abstract base classes (ABC).

## Materi

### Konsep Dasar
**Polymorphism**: Ability to use a unified interface for different types. Python achieves polymorphism through duck typing and inheritance.

**Duck Typing**: "If it walks like a duck and quacks like a duck, it must be a duck." Artinya, kita tidak perlu cek tipe formal; kita hanya perlu memastikan objek memiliki methods yang kita butuhkan.

```python
def make_sound(animal):
    print(animal.speak())  # asumsi animal punya method speak()
```

**Abstract Base Classes (ABC)**: Dari `abc` module, digunakan untuk define interface formal. ABC tidak bisa di-instansiasi, dan subclass harus implement abstract methods.

```python
from abc import ABC, abstractmethod

class Animal(ABC):
    @abstractmethod
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return "Woof"
```

### Polymorphism via Inheritance
Subclass bisa override methods. Function yang mengharapkan superclass bisa menerima subclass.

### Interface vs Implementation
- Duck typing: fokus pada behavior (methods yang tersedia), bukan tipe
- ABC: define kontrak formal, enforce implementasi
- Structural subtyping (Protocols) juga bisa digunakan (PEP 544)

### Detail Teknis
- `isinstance(obj, Class)` bisa digunakan tapi kadang tidak perlu dengan duck typing
- `hasattr(obj, 'method')` bisa cek apakah obj memiliki attribute (tapi lebih baik "easier to ask for forgiveness than permission" - EAFP)
- ABC: `@abstractmethod` harus di-override; jika tidak, subclass tetap abstract dan tidak bisa di-instansiasi
- ABC bisa memiliki concrete methods juga
- `ABC` bisa digunakan untuk mixins dan interfaces
- Python's polymorphism lebih fleksibel karena dynamic typing
