# Exception Chaining and Best Practices
**ID**: `M06-L04`  
**Type**: lesson

## Tujuan Pembelajaran
Preserve traceback with raise from, follow Pythonic error handling patterns.

## Materi

### Konsep Dasar
**Exception Chaining**: Ketika menangkap exception dan raise exception baru, gunakan `raise NewException(...) from original_exception` untuk mempertahankan traceback asli. Ini membantu debugging karena menunjukkan akar masalah.

```python
try:
    risky_operation()
except SomeError as e:
    raise MyError("Higher-level failure") from e
```

Output traceback akan menunjukkan Both exceptions:
```
Traceback (most recent call last):
  File "...", line ..., in ...
    SomeError: original
The above exception was the direct cause of the following exception:
Traceback (most recent call last):
  ...
MyError: Higher-level failure
```

**Best Practices**:
- Catch specific exceptions, not bare `except:` (kecuali really needed)
- Jangan swallow exceptions silently (minimal log atau raise)
- Gunakan custom exception untuk level abstraksi yang berbeda
- Sertakan informasi yang berguna dalam exception message
- `finally` untuk cleanup, `else` untuk success path
- Hindari exception untuk control flow (kecuali truly exceptional)

### Detail Teknis
- `raise from None` untuk men-suppress chain (tidak disarankan kecuali intentional)
- `sys.exc_info()` memberikan (type, value, traceback) saat dalam except block
- `logging.exception()` otomatis log traceback
- `except Exception as e:` menangkap semua built-in exceptions (tapi bukan KeyboardInterrupt/SystemExit)
- `raise` tanpa arguments di dalam except akan re-raise dengan traceback asli (tidak membuat chain baru)
- Defensive programming: validate input early, raise appropriate exceptions
