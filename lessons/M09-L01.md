## Materi

### Konsep Dasar
**Threading**: Eksekusi concurrent dalam satu process (threads berbagi memory). Berguna untuk I/O-bound tasks (network, file). Python memiliki GIL (Global Interpreter Lock) yang membatasi eksekusi parallel pada CPU-bound code.

**Creating Threads**:
```python
import threading

def worker():
    print("Working")

t = threading.Thread(target=worker, args=(...), kwargs={...})
t.start()   # start thread
t.join()    # wait until finished
```

**Daemon Threads**: Thread yang akan terminate otomatis saat program exit (jika masih hidup). `t.daemon = True` sebelum `start()`.

### Synchronization Primitives
- **Lock** (mutex): `lock.acquire()`, `lock.release()`, atau `with lock:` untuk critical section.
- **RLock** (reentrant lock): Thread yang sudah acquire bisa acquire lagi (recursive).
- **Event**: `event.set()`, `event.clear()`, `event.wait()`. Thread bisa menunggu event.
- **Semaphore**: `threading.Semaphore(value)` untuk batasan concurrent access.
- **Condition**: `condition.wait()`, `condition.notify()` untuk complex coordination.

### Global Interpreter Lock (GIL)
GIL memastikan hanya satu thread yang mengeksekusi Python bytecode pada satu waktu. Ini membuat threading tidak memberikan speedup untuk CPU-bound tasks (karena hanya satu thread yang benar-benar berjalan). Untuk I/O-bound, threading tetap efektif karena thread menunggu I/O sementara GIL dilepas.

### Detail Teknis
- `threading.current_thread()` mendapatkan current thread object
- `threading.main_thread()` untuk main thread
- `threading.enumerate()` list semua alive threads
- `t.is_alive()` cek status
- Race condition terjadi jika multiple threads akses shared data tanpa synchronization
- Deadlock: dua thread menunggu lock milik masing-masing
- Locking: `with lock:` lebih aman daripada `acquire/release` manual
- `threading.local()` untuk thread-local storage (data yang exclusive per thread)

## Latihan

### Soal 1
Buat program yang membuat 5 threads, masing-masing mencetak "Hello from thread X" di mana X adalah nomor thread (0-4). Gunakan `threading.Thread` dan `target` function yang menerima argumen. Tunggu semua thread selesai dengan `join()`.

```python
import threading
import time

def worker(num):
    time.sleep(0.1)  # simulasi work
    print(f"Hello from thread {num}")

def latihan_1():
    # Tulis kode kamu di sini
    pass

if __name__ == "__main__":
    latihan_1()
```

**Expected Output** (order mungkin tidak teratur):
```
Hello from thread 0
Hello from thread 1
Hello from thread 2
Hello from thread 3
Hello from thread 4
```

### Soal 2
Buat program dengan shared variable `counter = 0`. Buat 10 threads, masing-masing increment `counter` sebesar 1000 kali (total seharusnya 10000). Gunakan `Lock` untuk melindungi critical section. Tanpa lock, hasil akan acak (race condition). Verifikasi hasil akhir.

```python
import threading

counter = 0
lock = threading.Lock()

def increment():
    global counter
    # Tulis kode kamu di sini
    pass

def latihan_2():
    threads = []
    for _ in range(10):
        t = threading.Thread(target=increment)
        t.start()
        threads.append(t)
    for t in threads:
        t.join()
    print(f"Final counter: {counter}")

if __name__ == "__main__":
    latihan_2()
```

**Expected Output**:
```
Final counter: 10000
```
