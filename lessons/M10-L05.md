# Code Quality: PEP 8, Linters, and Formatters
**ID**: `M10-L05`  
**Type**: lesson  
**Duration**: 30-60 menit  
**Tags**: python, tutorial

## Tujuan Pembelajaran
Follow PEP 8, use flake8/pylint, and auto-format with black and isort.

## Materi

### Konsep Dasar
**PEP 8**: Style guide untuk Python code. Menjaga konsistensi readability.

**Key PEP 8 Rules**:
- Indentation: 4 spaces per level (no tabs)
- Maximum line length: 79 characters (PEP 8) or 99 (many modern projects)
- Blank lines: 2 between top-level functions/classes, 1 between methods
- Imports: grouped (standard library, third-party, local), each group separated by blank line; within group sorted alphabetically
- Naming: `snake_case` for functions/variables, `CamelCase` for classes, `UPPER_SNAKE_CASE` for constants
- Whitespace: around operators after commas, but not inside parentheses/brackets/braces
- Trailing commas in multi-line collections

**Linters**: Tools yang menganalisis code untuk potential errors dan style violations.
- **flake8**: combination of PyFlakes (errors), pycodestyle (PEP 8), McCabe (complexity)
- **pylint**: lebih verbose, lebih banyak checks (best practices, design, etc.)

**Formatters**: Auto-format code sesuai style guide.
- **black**: "uncompromising" formatter, opinionated, konsisten
- **isort**:专门 untuk sorting imports

### Using flake8
```bash
pip install flake8
flake8 my_module.py
```
Output: `file.py:line:col: code message`

### Using black
```bash
pip install black
black my_module.py  # modifies file in-place
black . --check  # check only, no change
```

### Using isort
```bash
pip install isort
isort my_module.py  # sort imports
```

### Pre-commit Hooks
Gunakan `pre-commit` framework untuk otomatis run linters/formatters sebelum commit.
```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/psf/black
    rev: 22.3.0
    hooks:
      - id: black
  - repo: https://github.com/pycqa/isort
    rev: 5.10.1
    hooks:
      - id: isort
  - repo: https://github.com/pycqa/flake8
    rev: 4.0.1
    hooks:
      - id: flake8
```
Install: `pre-commit install`, lalu otomatis setiap `git commit`.

### Detail Teknis
- flake8 config via `setup.cfg`, `tox.ini`, or `.flake8`
- pylint messages: `C` (convention), `R` (refactor), `W` (warning), `E` (error), `F` (fatal)
- black line length default 88 characters (bisa ubah dengan `--line-length`)
- black tidak mengubah semantics; hanya formatting
- isort profile: `black` profile untuk compatibility dengan black
- Bisa integrate dengan IDE (VS Code, PyCharm) untuk on-save formatting
- CI/CD pipelines: run linters dan formatters check

## Latihan

### Soal 1
Instal flake8 dan black (jika belum). Ambil file kode yang buruk style (berikan contoh), lalu:
- Jalankan `flake8` untuk melihat violations
- Perbaiki manual sesuai PEP 8
- Jalankan `black` untuk auto-format
- Verifikasi `flake8` lagi (harus lebih sedikit violations)

Contoh kode buruk:
```python
def add(a,b):
    return a+b
```

Setelah black: `def add(a, b):\n    return a + b`

**Tugas**: Tulis script yang sengaja violates PEP 8 (misal: long line, bad spacing, mixed imports), lalu perbaiki dengan tools.

```bash
# commands to run
flake8 bad_style.py
black bad_style.py
flake8 bad_style.py
```

**Expected Output**: flake8 pertama shows violations, setelah black shows none (or fewer).

### Soal 2
Buat file module dengan imports dari standard library, third-party, dan local. Lalu:
- Jalankan `isort` untuk sort imports dengan profile black
- Cek urutan: standard (alphabetical), blank line, third-party (alphabetical), blank line, local (alphabetical)
- Buat `.isort.cfg` atau `pyproject.toml` untuk konfigurasi

```python
# mymodule.py (sebelum isort)
import os
import sys
import numpy as np
from mypackage import local_module
import requests
from datetime import datetime

def main():
    pass
```

Setelah isort:
```python
import os
import sys
from datetime import datetime

import numpy as np
import requests

from mypackage import local_module
```

**Tugas**: Jalankan isort, periksa hasilnya.

**Expected Output**: imports terurut sesuai grup dan alphabet.
