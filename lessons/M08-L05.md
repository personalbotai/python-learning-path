# Descriptors and the Descriptor Protocol
**ID**: `M08-L05`  
**Type**: lesson  
**Duration**: 30-60 menit  
**Tags**: python, tutorial

## Tujuan Pembelajaran
Implement __get__, __set__, __delete__ for custom attribute behavior.

## Materi

### Konsep Dasar
**Descriptor**: Objek yang mengimplementasikan salah satu dari `__get__`, `__set__`, atau `__delete__`. Digunakan untuk custom attribute access pada class.

**Types**:
- **Data descriptor**: Mengimplementasikan `__set__` dan `__get__` (dan/atau `__delete__`). Memiliki precedence lebih tinggi daripada instance dictionary.
- **Non-data descriptor**: Hanya `__get__`. Precedence lebih rendah; instance attribute akan menimpa.

**Syntax**:
```python
class Descriptor:
    def __init__(self, name):
        self.name = name

    def __get__(self, instance, owner):
        # instance: objek yang memiliki attribute, atau None jika diakses via class
        # owner: class yang own descriptor
        return instance.__dict__[self.name]

    def __set__(self, instance, value):
        instance.__dict__[self.name] = value

    def __delete__(self, instance):
        del instance.__dict__[self.name]

class MyClass:
    attr = Descriptor('attr')
```

### Use Cases
- Type checking
- Lazy loading (computed on first access)
- Validation
- Logging attribute access
- Read-only attributes
- Unit conversion (misal: Celsius/Fahrenheit)

### Property vs Descriptor
`@property` sebenarnya adalah data descriptor (hanya read-only jika tidak ada setter). Descriptor lebih general dan bisa digunakan di banyak attribute sekaligus.

### Detail Teknis
- Descriptor di-class dictionary, bukan instance dictionary
- `instance.__dict__` menyimpan actual data
- `owner` param sering tidak digunakan kecuali untuk class-level info
- Jika `__get__` diakses via class (`MyClass.attr`), `instance` adalah None
- Bisa membuat descriptor reusable dengan passing attribute name di `__init__` atau menggunakan `__set_name__(self, owner, name)` (Python 3.6+)
- `__set_name__` otomatis dipanggil saat class dibuat; name descriptor disimpan

## Latihan

### Soal 1
Buat descriptor `Typed` yang melakukan type checking. Saat set, jika value bukan tipe yang diharapkan (passed saat konstruksi), raise TypeError.

```python
class Typed:
    def __init__(self, name, expected_type):
        self.name = name
        self.expected_type = expected_type

    def __get__(self, instance, owner):
        # Tulis kode kamu di sini
        pass

    def __set__(self, instance, value):
        # Tulis kode kamu di sini
        pass

class Person:
    age = Typed('age', int)
    height = Typed('height', float)

    def __init__(self, age, height):
        self.age = age
        self.height = height

if __name__ == "__main__":
    p = Person(25, 1.75)
    print(p.age)   # 25
    p.age = "old"  # TypeError
```

**Expected Output**:
```
25
TypeError: Expected <class 'int'>, got <class 'str'>
```

### Soal 2
Buat descriptor `LazyAttribute` yang melakukan lazy computation: first access memanggil fungsi factory (diberikan saat konstruksi) untuk menghitung nilai, lalu menyimpan di instance dictionary agar tidak dihitung ulang.

```python
class LazyAttribute:
    def __init__(self, func):
        self.func = func
        self.name = func.__name__

    def __get__(self, instance, owner):
        # Tulis kode kamu di sini
        pass

class Data:
    @LazyAttribute
    def expensive_data(self):
        print("Computing...")
        return sum(range(1000000))

if __name__ == "__main__":
    d = Data()
    print("Accessing first time:")
    print(d.expensive_data)  # print "Computing..." lalu nilai
    print("Accessing second time:")
    print(d.expensive_data)  # tidak print "Computing..." lagi
```

**Expected Output**:
```
Accessing first time:
Computing...
499999500000
Accessing second time:
499999500000
```
