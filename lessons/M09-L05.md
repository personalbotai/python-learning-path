# Choosing the Right Concurrency Model
**ID**: `M09-L05`  
**Type**: lesson  
**Duration**: 30-60 menit  
**Tags**: python, tutorial

## Tujuan Pembelajaran
Decide between threading, multiprocessing, and asyncio based on I/O vs CPU bound tasks.

## Materi

### Konsep Dasar
Python menawarkan beberapa model concurrency:
- **Threading**: threads dalam satu process, GIL membatasi CPU-bound parallelism, baik untuk I/O-bound.
- **Multiprocessing**: multiple processes, bypass GIL, baik untuk CPU-bound.
- **Asyncio**: single-threaded event loop, cooperative multitasking, sangat baik untuk I/O-bound dengan many concurrent connections.

**Decision Criteria**:
1. **Task Type**:
   - I/O-bound (network, disk, database): Threading atau Asyncio
   - CPU-bound (komputasi, processing): Multiprocessing
2. **Number of Concurrent Tasks**:
   - Many (hundreds/thousands) I/O operations: Asyncio lebih efisien (low overhead)
   - Moderate ( few dozen) I/O operations: Threading cukup
3. **Legacy Code**:
   - Jika library blocking (tidak support async), threading lebih mudah
4. **Complexity**:
   - Asyncio require async/await pattern; bisa rumit untuk debugging
   - Threading lebih familiar tapi ada race conditions
5. **Memory Usage**:
   - Threads share memory (tapi perlu lock)
   - Processes terisolasi (lebih memory tapi lebih safe)

### Comparison Table

| Feature | Threading | Multiprocessing | Asyncio |
|---------|-----------|-----------------|---------|
| Parallelism | No (GIL) | Yes (multiple processes) | No (cooperative) |
| Best for | I/O-bound | CPU-bound | I/O-bound, many connections |
| Overhead | Moderate | High (process creation) | Low |
| Memory | Shared | Separate | Shared (single thread) |
| Complexity | Moderate | Moderate | Higher (async/await) |
| Debugging | Hard (races) | Easier (isolated) | Hard (event loop) |

### When to Use What
- **Web scraper dengan requests (blocking)**: Threading (simple) atau Asyncio dengan aiohttp (advanced)
- **Data processing (komputasi berat)**: Multiprocessing
- **Chat server / real-time web**: Asyncio
- **GUI aplikasi dengan background tasks**: Threading (jaga UI responsiveness)
- **Mixing CPU & I/O**: Combine (misal: asyncio untuk I/O, process pool untuk CPU)

### Hybrid Approaches
Bisa menggabungkan: `asyncio` dengan `run_in_executor()` untuk menjalankan blocking code di thread pool atau process pool.
```python
loop = asyncio.get_event_loop()
result = await loop.run_in_executor(None, blocking_func, arg)
```

### Detail Teknis
- Thread safe: data structures seperti `queue.Queue` thread-safe; list/dict tidak
- Process safe: `multiprocessing.Queue`, `Pipe`, `Manager`
- Asyncio: semua I/O operations harus async (library support)
- GIL dilepas saat I/O wait atau explicit release (`time.sleep` juga release GIL)
- `concurrent.futures` bisa abstraksi: `ThreadPoolExecutor` atau `ProcessPoolExecutor` dengan API yang sama
- `asyncio` tidak cocok untuk CPU-bound tasks (tidak parallelism)
