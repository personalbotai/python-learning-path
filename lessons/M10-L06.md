## Tujuan Pembelajaran

Setelah mengerjakan pelajaran ini, kamu akan mampu:

- memahami standard Python project structure
- menggunakan virtual environments (`venv`, `pipenv`, `poetry`)
- menulis `setup.py` atau `pyproject.toml` untuk packaging
- menerapkan PEP 8 style guide dan linters (black, flake8, isort)
- menulis comprehensive README dan documentation
- menggunakan Git untuk version control dengan branching strategy (Git Flow)
- memahami semantic versioning

## Materi

### Konsep Dasar

**Project structure** adalah how you organize files and directories dalam project. Good structure membuat code maintainable, scalable, dan mudah dipahami oleh tim.

**Best practices** adalah kumpulan aturan dan konvensi yang terbukti meningkatkan kualitas software.

### Detail Teknis

**2. Virtual Environments**

```bash
# Create venv (built-in)
python -m venv .venv

# Activate
# Linux/macOS:
source .venv/bin/activate
# Windows:
.venv\Scripts\activate

# Deactivate
deactivate

# Install dependencies
pip install -r requirements.txt

# pipenv (higher-level)
pipenv install requests
pipenv install --dev pytest black

# poetry (modern, lockfile)
poetry add requests
poetry add --group dev pytest
```

**3. `pyproject.toml` (Modern Standard)**

```toml
[build-system]
requires = ["setuptools>=61.0", "wheel"]
build-backend = "setuptools.build_meta"

[project]
name = "mypackage"
version = "0.1.0"
description = "My awesome Python package"
readme = "README.md"
authors = [{name = "Alice", email = "alice@example.com"}]
license = {text = "MIT"}
classifiers = [
    "Development Status :: 3 - Alpha",
    "Intended Audience :: Developers",
    "Programming Language :: Python :: 3",
    "Programming Language :: Python :: 3.9",
]
keywords = ["example", "package"]
requires-python = ">=3.9"
dependencies = [
    "requests>=2.25.0",
    "click>=7.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=6.0",
    "black>=21.0",
    "flake8>=3.9",
    "mypy>=0.910",
]

[project.scripts]
mytool = "mypackage.cli:main"

[tool.black]
line-length = 88
target-version = ['py39']

[tool.isort]
profile = "black"

[tool.pytest.ini_options]
testpaths = ["tests"]
python_files = "test_*.py"
```

**4. `setup.py` (Legacy, still works)**

```python
from setuptools import setup, find_packages

setup(
    name="mypackage",
    version="0.1.0",
    packages=find_packages(where="src"),
    package_dir={"": "src"},
    install_requires=[
        "requests>=2.25.0",
    ],
    extras_require={
        "dev": ["pytest", "black"],
    },
    entry_points={
        "console_scripts": [
            "mytool=mypackage.cli:main",
        ],
    },
    author="Alice",
    description="My package",
    python_requires=">=3.9",
)
```

**5. PEP 8 & Code Quality Tools**

```bash
# black (opinionated code formatter)
black src/ tests/ --line-length 88

# isort (import sorting)
isort src/ tests/

# flake8 (linter)
flake8 src/ tests/ --max-line-length=88 --ignore=E203,W503

# mypy (type checker)
mypy src/

# pre-commit hooks (automate before git commit)
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/psf/black
    rev: 22.3.0
    hooks:
      - id: black
  - repo: https://github.com/pycqa/isort
    rev: 5.10.1
    hooks:
      - id: isort
  - repo: https://github.com/pycqa/flake8
    rev: 4.0.1
    hooks:
      - id: flake8
```

**6. Git & Version Control**

```bash
# Initialize repo
git init
git add .
git commit -m "Initial commit"

# Branching strategy (Git Flow)
# main (production-ready)
# develop (integration branch)
# feature/xxx (new feature)
# bugfix/xxx (bug fix)
# release/v1.0.0 (release prep)
# hotfix/xxx (critical fix on production)

# Example workflow
git checkout -b feature/user-auth
# ... work ...
git add .
git commit -m "Add user authentication"
git push origin feature/user-auth
# Create PR to develop

# Semantic Versioning (SemVer)
# MAJOR.MINOR.PATCH (1.2.3)
# - MAJOR: incompatible API changes
# - MINOR: add functionality in backward-compatible manner
# - PATCH: backward-compatible bug fixes
```

**8. Continuous Integration (CI)**

```yaml
# .github/workflows/tests.yml
name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.9", "3.10", "3.11"]
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v4
        with:
          python-version: ${{ matrix.python-version }}
      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -e .[dev]
      - name: Run tests
        run: pytest -v --cov=src --cov-report=xml
      - name: Upload coverage
        uses: codecov/codecov-action@v3
```

**9. Environment Configuration**

```python
# config.py
import os
from dataclasses import dataclass
from typing import Optional

@dataclass
class Config:
    debug: bool
    database_url: str
    redis_url: Optional[str] = None
    log_level: str = "INFO"

def load_config() -> Config:
    return Config(
        debug=os.getenv("DEBUG", "False").lower() == "true",
        database_url=os.getenv("DATABASE_URL", "sqlite:///dev.db"),
        redis_url=os.getenv("REDIS_URL"),
        log_level=os.getenv("LOG_LEVEL", "INFO"),
    )
```

## Latihan

### Soal 1: Create Project Structure

Buat struktur project untuk "Task Manager API":

```
taskmanager/
в”ңв”Җв”Җ pyproject.toml
в”ңв”Җв”Җ README.md
в”ңв”Җв”Җ .gitignore
в”ңв”Җв”Җ requirements.txt (optional)
в”ңв”Җв”Җ src/
в”Ӯ   в””в”Җв”Җ taskmanager/
в”Ӯ       в”ңв”Җв”Җ __init__.py
в”Ӯ       в”ңв”Җв”Җ api.py          # FastAPI/Flask app
в”Ӯ       в”ңв”Җв”Җ models.py       # Pydantic models / SQLAlchemy
в”Ӯ       в”ңв”Җв”Җ crud.py         # Database operations
в”Ӯ       в”ңв”Җв”Җ database.py     # DB connection
в”Ӯ       в””в”Җв”Җ config.py       # Configuration
в”ңв”Җв”Җ tests/
в”Ӯ   в”ңв”Җв”Җ __init__.py
в”Ӯ   в”ңв”Җв”Җ conftest.py
в”Ӯ   в”ңв”Җв”Җ test_api.py
в”Ӯ   в””в”Җв”Җ test_crud.py
в”ңв”Җв”Җ scripts/
в”Ӯ   в””в”Җв”Җ init_db.py
в”ңв”Җв”Җ .env.example
в””в”Җв”Җ .github/workflows/tests.yml
```

**Task:**
1. Buat `pyproject.toml` dengan dependencies: `fastapi`, `uvicorn`, `sqlalchemy`, `pydantic`, `pytest`
2. Buat `README.md` dengan installation dan usage
3. Buat `.gitignore` untuk Python (virtual env, __pycache__, .pyc, .env, .coverage, .pytest_cache, dist, build, *.egg-info)
4. Buat `config.py` yang load dari environment variables
5. Buat `database.py` dengan SQLAlchemy engine
6. Buat `models.py` dengan User, Task models
7. Buat `crud.py` dengan CRUD operations
8. Buat `api.py` dengan FastAPI routes
9. Buat `tests/test_api.py` dengan basic API tests
10. Buat `.github/workflows/tests.yml` untuk CI

### Soal 2: Write Comprehensive README

Untuk project di soal 1, tulis README yang mencakup:

- Project description
- Features
- Installation (with pip/poetry)
- Configuration (environment variables)
- Usage (how to run API, example requests with curl)
- Testing (how to run tests)
- API documentation (mention /docs otomatis FastAPI)
- Contributing guidelines
- License (MIT)

### Soal 3: Setup Pre-commit Hooks

Buat `.pre-commit-config.yaml` untuk project:

```yaml
# TODO: isi dengan hooks:
# - black (formatting)
# - isort (import sorting)
# - flake8 (linting)
# - mypy (type checking)
# - pytest (run tests)
# Gunakan `pre-commit install` untuk setup
```

Buat juga `Makefile` untuk common tasks:

```makefile
.PHONY: install test lint format type-check

install:
    pip install -e .[dev]

test:
    pytest -v --cov=src

lint:
    flake8 src tests

format:
    black src tests
    isort src tests

type-check:
    mypy src
```
