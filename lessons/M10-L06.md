## Tujuan Pembelajaran
Run static type checking, understand common mypy errors, improve code reliability.

## Materi

### Konsep Dasar
**mypy**: Static type checker untuk Python. Memeriksa kode berdasarkan type hints (PEP 484) tanpa menjalankannya. Menemukan potential type errors sebelum runtime.

**Installation**:
```bash
pip install mypy
```

**Basic Usage**:
```bash
mypy my_module.py
```
Output: error messages dengan baris dan kolom.

**Type Hints**: Tambahkan `: type` untuk parameters dan `-> type` untuk return.
```python
def greet(name: str) -> str:
    return "Hello " + name
```

### Common mypy Errors
- **Incompatible types**: `Argument 1 to "foo" has incompatible type "int"; expected "str"`
- **Missing attributes**: `Item "int" of "Union[int, str]" has no attribute "upper"`
- **Invalid return type**: `Missing type parameters for generic type "List"`
- **Unused ignore**: `error: unused "type: ignore" comment`
- **Name undefined**: `Name 'x' is not defined`

### Configuring mypy
File `mypy.ini` atau `pyproject.toml`:
```ini
[mypy]
python_version = 3.9
warn_return_any = True
warn_unused_configs = True
disallow_untyped_defs = True
check_untyped_defs = True
```
Atau command-line flags: `mypy --strict my_module.py`.

### Type Hinting Tips
- Gunakan `from typing import List, Dict, Optional, Union, Callable, Any`
- Python 3.9+: `list[int]`, `dict[str, int]` (built-in collection types)
- `Optional[T]` sama dengan `Union[T, None]`
- `Any` menonaktifkan checking (hindari jika bisa)
- `cast(type, value)` untuk tell mypy about type you know
- `TypedDict` untuk dictionary dengan specific keys
- `Protocol` untuk structural subtyping (duck typing with types)

### Detail Teknis
- mypy check function bodies, tapi tidak mengeksekusi kode
- Bisa ignore error dengan `# type: ignore` (tapi sebaiknya hindari)
- `--ignore-missing-imports` untuk packages tanpa stubs
- Third-party packages mungkin perlu type stubs (`types-requests`, dll)
- `mypy` bisa diintegrasikan ke CI pipeline
- `reveal_type(expr)` akan menampilkan type yang di-infer oleh mypy (debugging type hints)
- `# mypy: follow-imports=skip` untuk ignore imported modules

## Latihan

### Soal 1
Buat file `calculator.py` dengan fungsi-fungsi matematika sederhana (tambahan, pengurangan, perkalian, pembagian) dengan type hints (parameter `float`, return `float`). Tambahkan juga fungsi `average(nums: list[float]) -> float`. Lalu jalankan `mypy` pada file itu. Perbaiki semua errors (misal: penggunaan `int` di mana `float` diharapkan, atau return `None`).

```python
# calculator.py
def add(a: float, b: float) -> float:
    return a + b

def subtract(a: float, b: float) -> float:
    return a - b

def multiply(a: float, b: float) -> float:
    return a * b

def divide(a: float, b: float) -> float:
    if b == 0:
        return None  # mypy akan complain karena None vs float
    return a / b

def average(nums: list) -> float:
    return sum(nums) / len(nums)

if __name__ == "__main__":
    print(average([1, 2, 3]))
```

**Tugas**: Jalankan `mypy calculator.py`. Perbaiki errors:
- `divide` harus return `float` atau raise exception, jangan `None`
- `average` parameter harus `list[float]` (atau `Sequence[float]`)
- Pastikan semua type hints konsisten.

**Expected Output** setelah perbaikan:
```
Success: no issues found in 1 source file
```

### Soal 2
Buat module `data_processor.py` dengan fungsi yang menggunakan `TypedDict` dan `Optional`. Contoh: fungsi `process_user(user: UserDict) -> str` dengan `UserDict = TypedDict('UserDict', {'name': str, 'age': int})`. Juga fungsi `find_user(users: list[UserDict], name: str) -> Optional[UserDict]`. Tambahkan type hints, lalu cek dengan mypy.

```python
from typing import TypedDict, Optional, List

User = TypedDict('User', {'name': str, 'age': int})

def process_user(user: User) -> str:
    return f"{user['name']} is {user['age']} years old"

def find_user(users: List[User], name: str) -> Optional[User]:
    for u in users:
        if u['name'] == name:
            return u
    return None

if __name__ == "__main__":
    users = [{'name': 'Alice', 'age': 25}, {'name': 'Bob', 'age': 30}]
    print(process_user(users[0]))
    print(find_user(users, 'Charlie'))  # None
```

**Tugas**: Jalankan `mypy data_processor.py`. Harus `Success`.

**Expected Output**:
```
Success: no issues found in 1 source file
```
