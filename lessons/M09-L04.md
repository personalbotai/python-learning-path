## Materi

### Konsep Dasar
**concurrent.futures**: Module yang menyediakan interface tingkat tinggi untuk asynchronous execution. Dua executor utama:
- `ThreadPoolExecutor`: untuk I/O-bound tasks (threads)
- `ProcessPoolExecutor`: untuk CPU-bound tasks (processes)

**Executor Interface**:
```python
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor
import time

def task(n):
    time.sleep(1)
    return n * n

with ThreadPoolExecutor(max_workers=4) as executor:
    # Submit single task
    future = executor.submit(task, 5)
    result = future.result()  # blocking

    # Map multiple tasks
    results = executor.map(task, [1,2,3,4,5])
    for r in results:
        print(r)
```

**Future Objects**: `Future` mewakili eksekusi yang belum selesai. Method: `result()`, `exception()`, `add_done_callback(fn)`, `cancel()`.

### ThreadPoolExecutor vs ProcessPoolExecutor
- `ThreadPoolExecutor`: Shared memory, GIL berlaku, baik untuk I/O-bound
- `ProcessPoolExecutor`: Terpisah memory, bypass GIL, baik untuk CPU-bound, tetapi serialization overhead (pickle)

### Best Practices
- Gunakan context manager (`with`) untuk otomatis shutdown executor
- `max_workers`: default = number of processors * 5 untuk ThreadPoolExecutor; untuk ProcessPoolExecutor biasanya jumlah CPU cores
- `executor.map()` mengembalikan iterator hasil dalam urutan input
- `as_completed(futures)` untuk iterate saat task selesai (tidak urut)
- `timeout` parameter di `future.result(timeout=...)` untuk avoid hung

### Detail Teknis
- `submit()` mengembalikan `Future` untuk satu task
- `map()` batch tasks, return iterator; raises exception saat iterate jika task error (jika ingin tangkap, iterate dengan try/except)
- `executor.shutdown(wait=True)` secara otomatis dipanggil oleh `__exit__` context manager
- Bisa `executor.submit(fn, *args, **kwargs)` dengan arbitrary arguments
- `future.done()` cek apakah selesai
- `future.exception()` mengembalikan exception jika ada, else None
- `concurrent.futures.wait(futures)` untuk menunggu multiple futures

### Soal 1
Gunakan `ThreadPoolExecutor` untuk download simulasi (task yang sleep 1 detik) untuk 5 URLs secara concurrent. Gunakan `executor.map()` dan cetak hasil.ukur waktu eksekusi total dengan `time.time()`.

```python
from concurrent.futures import ThreadPoolExecutor
import time

def download(url):
    time.sleep(1)
    return f"Content from {url}"

def latihan_1():
    urls = [f"http://site{i}.com" for i in range(5)]
    start = time.time()
    # Tulis kode kamu di sini dengan ThreadPoolExecutor
    end = time.time()
    print(f"Total time: {end-start:.2f}s")

if __name__ == "__main__":
    latihan_1()
```

**Expected Output**:
```
Content from http://site0.com
Content from http://site1.com
Content from http://site2.com
Content from http://site3.com
Content from http://site4.com
Total time: ~1.0s (not 5s)
```
