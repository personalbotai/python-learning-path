## Materi

### Konsep Dasar

Errors dalam Python dibagi menjadi:
- **Syntax Errors**: errors saat parsing (tidak bisa di-handle)
- **Exceptions**: runtime errors yang bisa di-handle dengan try-except

**Basic try-except:**
```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Tidak bisa bagi dengan nol!")

# Atau capture exception object
try:
    with open("nonexistent.txt") as f:
        content = f.read()
except FileNotFoundError as e:
    print(f"File tidak ditemukan: {e}")
    # e.filename, e.strerror, dll
```

### Detail Teknis

**1. Exception Hierarchy**
```
BaseException
 ├── SystemExit
 ├── KeyboardInterrupt
 ├── GeneratorExit
 └── Exception
      ├── StopIteration
      ├── ArithmeticError (ZeroDivisionError, OverflowError, FloatingPointError)
      ├── LookupError (IndexError, KeyError)
      ├── ValueError (UnicodeError, etc.)
      ├── TypeError
      ├── RuntimeError
      ├── OSError (FileNotFoundError, PermissionError, etc.)
      ├── MemoryError
      └── ... (banyak lagi)
```

**Catch specific vs general:**
```python
try:
    # code
except ValueError:       # spesifik
    print("Invalid value")
except TypeError:         # spesifik
    print("Wrong type")
except Exception:        # general (catch semua except system-exiting)
    print("Something went wrong")
# except:                # super general (catch BaseException, termasuk SystemExit)
#     print("Everything")
```

**2. try-except-else-finally**
```python
try:
    result = risky_operation()
except SomeError as e:
    # Handle error
    print(f"Error: {e}")
else:
    # Hanya executed jika tidak ada exception
    print(f"Result: {result}")
finally:
    # Selalu executed (cleanup)
    cleanup_resources()
```

**3. Multiple Exceptions**
```python
try:
    x = int(input("Enter number: "))
    y = 10 / x
except (ValueError, TypeError):
    print("Invalid input type")
except ZeroDivisionError:
    print("Cannot divide by zero")
except Exception as e:
    print(f"Unexpected error: {e}")
```

**4. Raising Exceptions**
```python
def validate_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative")
    if age > 150:
        raise ValueError("Age seems unrealistic")
    return True

try:
    validate_age(-5)
except ValueError as e:
    print(e)  # "Age cannot be negative"

# Re-raise exception
def process():
    try:
        do_something()
    except Exception:
        print("Logging error...")
        raise  # re-raise same exception
```

**5. Custom Exceptions**
```python
class MyAppError(Exception):
    """Base exception for my app"""
    pass

class ValidationError(MyAppError):
    """Raised when validation fails"""
    def __init__(self, field, message):
        self.field = field
        self.message = message
        super().__init__(f"{field}: {message}")

class InsufficientFundsError(MyAppError):
    """Raised when balance too low"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"Balance {balance} insufficient for withdrawal {amount}")

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    withdraw(100, 150)
except InsufficientFundsError as e:
    print(e)  # "Balance 100 insufficient for withdrawal 150"
    print(f"Can only withdraw up to {e.balance}")
```

**6. Assert Statements (debugging)**
```python
def calculate_average(nums):
    assert len(nums) > 0, "nums cannot be empty"  # raises AssertionError jika False
    return sum(nums) / len(nums)

# Run with -O (optimize) flags, assertions di-skip
# python -O script.py -> assertions tidak aktif
```

**7. Exception Chaining (Python 3)**
```python
def read_config():
    try:
        with open("config.json") as f:
            return json.load(f)
    except FileNotFoundError as e:
        raise ConfigurationError("Config file missing") from e
        # "from e" chain ke original exception
```

**8. Context Managers & With Statement** (will be covered in next lesson, but related)
```python
# with statement auto handles cleanup (__enter__, __exit__)
with open("file.txt") as f:
    content = f.read()
# file automatically closed, even if exception occurs

# Custom context manager
from contextlib import contextmanager

@contextmanager
def managed_file(filename):
    f = open(filename, "w")
    try:
        yield f
    finally:
        f.close()

with managed_file("hello.txt") as f:
    f.write("Hello")
# file closed automatically
```

### Soal 3: Retry Decorator

Buat decorator `retry(max_attempts=3, delay=1)` yang:
1. Menjalankan function, jika exception tertentu (misal: ConnectionError, TimeoutError) terjadi, wait `delay` seconds lalu retry
2. Jika setelah `max_attempts` masih gagal, raise exception terakhir
3. Jika sukses, return result

```python
import time
from functools import wraps

def retry(max_attempts=3, delay=1, exceptions=(Exception,)):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts:
                        raise
                    print(f"Attempt {attempt} failed: {e}. Retrying in {delay}s...")
                    time.sleep(delay)
        return wrapper
    return decorator

@retry(max_attempts=3, delay=0.1, exceptions=(ValueError,))
def flaky_function(success_on_attempt=2):
    """Simulate function yang gagal 2 kali, lalu sukses"""
    flaky_function.attempt = getattr(flaky_function, 'attempt', 0) + 1
    if flaky_function.attempt < success_on_attempt:
        raise ValueError("Simulated failure")
    return "Success!"

def main():
    try:
        result = flaky_function(success_on_attempt=3)
        print(f"Result: {result}")
    except ValueError as e:
        print(f"Final failure: {e}")
```
