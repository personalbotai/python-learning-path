## Tujuan Pembelajaran
Implement __str__, __repr__, __len__, __getitem__, __setitem__, and other magic methods.

## Materi

### Konsep Dasar
Dunder (double underscore) methods adalah special methods yang mengizinkan class Python berperilaku seperti built-in types. Dikenali dengan nama `__nama__`.

**Common Dunder Methods**:
- `__str__(self)`: string untuk user (print(obj))
- `__repr__(self)`: string untuk developer (representasi unambiguous, ideal untuk debugging)
- `__len__(self)`: untuk `len(obj)` (harus return int >= 0)
- `__getitem__(self, key)`: untuk `obj[key]` (indexing)
- `__setitem__(self, key, value)`: untuk `obj[key] = value`
- `__delitem__(self, key)`: untuk `del obj[key]`
- `__iter__(self)`: untuk iterasi (return iterator)
- `__next__(self)`: untuk iterator (return next value atau raise StopIteration)
- `__call__(self, ...)`: membuat instance bisa dipanggil seperti fungsi
- Arithmetic: `__add__`, `__sub__`, `__mul__`, `__truediv__`, dll
- Comparison: `__eq__`, `__ne__`, `__lt__`, `__le__`, `__gt__`, `__ge__`

### Detail Teknis
- `__str__` untuk display; `__repr__` untuk unambiguous representation (bisa eval'd jika memungkinkan)
- Jika `__str__` tidak didefinisikan, Python fallback ke `__repr__`
- `__len__` harus return integer non-negative; digunakan oleh `len()` dan untuk truthiness (`__bool__` fallback ke `__len__`)
- `__getitem__` harus menangani slice objects (`slice`) juga, tidak hanya integer
- `__iter__` harus mengembalikan objek yang memiliki `__next__` (iterator)
- `__call__` membuat instance menjadi callable
- Dunder methods dipanggil oleh interpreter, bukan langsung dipanggil (kecuali intentional)
- Bisa override operator dengan dunder methods (operator overloading)

## Latihan

### Soal 1
Buat class `SimpleVector` yang:
- `__init__(self, components)` menerima list angka
- `__len__(self)` mengembalikan panjang vector (jumlah komponen)
- `__getitem__(self, index)` mengembalikan komponen pada index (support indexing dan slicing)
- `__str__(self)` mengembalikan representasi seperti `Vector(1, 2, 3)`
- `__repr__(self)` mengembalikan `"Vector([1, 2, 3])"` (bisa eval'd)

```python
class SimpleVector:
    # Tulis kode kamu di sini
    pass

if __name__ == "__main__":
    v = SimpleVector([1, 2, 3])
    print(len(v))       # 3
    print(v[0])         # 1
    print(v[1:])        # [2, 3]
    print(str(v))       # Vector(1, 2, 3)
    print(repr(v))      # Vector([1, 2, 3])
```

**Expected Output**:
```
3
1
[2, 3]
Vector(1, 2, 3)
Vector([1, 2, 3])
```

### Soal 2
Buat class `BankAccount` (lanjutan) dengan dunder methods:
- `__str__`: "Account(holder='Alice', balance=100.0)"
- `__repr__`: "BankAccount('Alice', 100.0)"
- `__add__(self, other)`: jika other adalah number, tambah balance dan kembalikan new BankAccount (immutable operation, jangan modifikasi asli); jika other adalah BankAccount, gabungkan balance (sum) dan gabungkan nama (contoh: "Alice & Bob")
- `__eq__(self, other)`: dua account equal jika holder sama dan balance sama

```python
class BankAccount:
    def __init__(self, holder, balance):
        self.holder = holder
        self.balance = balance

    # Tulis dunder methods di sini
    pass

if __name__ == "__main__":
    a1 = BankAccount("Alice", 100)
    a2 = BankAccount("Bob", 200)
    a3 = BankAccount("Alice", 100)
    print(a1)                # Account(holder='Alice', balance=100)
    print(repr(a2))          # BankAccount('Bob', 200)
    a4 = a1 + 50
    print(a4.balance)       # 150
    a5 = a1 + a2
    print(a5.holder)        # Alice & Bob
    print(a5.balance)       # 300
    print(a1 == a3)         # True
    print(a1 == a2)         # False
```

**Expected Output**:
```
Account(holder='Alice', balance=100)
BankAccount('Bob', 200)
150
Alice & Bob
300
True
False
```
