## Materi

### Konsep Dasar

Python memiliki 4 built-in collection types utama:

| Type | Ordered? | Mutable? | Unique? | Indexing? | Use Case |
|------|----------|----------|---------|-----------|----------|
| **List** | ✅ Yes | ✅ Yes | ❌ No | ✅ Yes | koleksi seragam, perlu modifikasi |
| **Tuple** | ✅ Yes | ❌ No | ❌ No | ✅ Yes | data tetap, multiple return, dict keys |
| **Dict** | ✅ (3.7+) | ✅ Yes | Keys: ✅ | ❌ No | lookup by key, associative array |
| **Set** | ❌ No | ✅ Yes | ✅ Yes | ❌ No | uniqueness, membership test, set ops |

### Detail Teknis

**1. Time Complexity (approximate)**
```python
# LIST (array-based)
my_list = [1,2,3,4,5]
my_list.append(10)      # O(1) - append
my_list.insert(0, 0)    # O(n) - insert di awal
my_list.pop()           # O(1) - pop last
my_list.pop(0)          # O(n) - pop first
my_list[3]              # O(1) - indexing
value in my_list        # O(n) - membership test (linear search)
my_list.sort()          # O(n log n)

# TUPLE (immutable list)
my_tuple = (1,2,3)
my_tuple[1]             # O(1) - indexing
value in my_tuple       # O(n) - membership test

# DICT (hash table)
my_dict = {"a":1, "b":2}
my_dict["c"] = 3        # O(1) - average insert
my_dict["a"]            # O(1) - lookup
my_dict.pop("a")        # O(1) - delete
"key" in my_dict        # O(1) - membership test

# SET (hash table, seperti dict tapi tanpa value)
my_set = {1,2,3}
my_set.add(4)           # O(1) - add
my_set.remove(2)        # O(1) - remove
value in my_set         # O(1) - membership test
set1 & set2             # O(min(len(s1), len(s2))) - intersection
```

**2. Memory Usage (rough order)**
```
Tuple < List < Dict ≈ Set
```
- Tuple paling hemat karena immutable, tidak perlu overhead untuk modification
- List perlu capacity buffer untuk growth
- Dict & Set perlu hash table (lebih banyak memory) tapi memberikan O(1) lookup

**3. Decision Matrix**

| Scenario | Recommended | Why |
|----------|-------------|-----|
| Data yang tidak akan berubah (koordinat, warna RGB) | Tuple | Immutable protection, bisa jadi dict key |
| Multiple return values dari fungsi | Tuple | Convention, unpacking mudah |
| Koleksi item yang sama jenis (user IDs, scores) | List | Ordered, mutable, indexable |
| Data dengan field bernama (person: name, age, city) | Dict | Key-based lookup, readable |
| Cek membership sering (whitelist, visited nodes) | Set | O(1) lookup vs O(n) list |
| Hapus duplikat dari list | Set | `list(set(mylist))` cepat |
| Gabungan/irisan koleksi (tags, categories) | Set | Union, intersection built-in |
| Data yang perlu diurutkan berdasarkan key | Dict (3.7+) | Maintains insertion order |
| Sequence yang perlu di-slice | List/Tuple | Support slicing |
| Data yang butuh sorted traversal | List (sort) atau sorted(dict.items()) |

**4. Common Pitfalls**

```python
# 1. Mutable default argument (BURUK!)
def process_items(items=[]):  # ❌ default mutable
    items.append("new")
    return items

# 2. Gunakan None sebagai default
def process_items(items=None):  # ✅
    if items is None:
        items = []

# 3. Shallow copy陷阱
list1 = [[1,2], [3,4]]
list2 = list1.copy()  # shallow copy
list2[0].append(99)   # list1[0] juga ikut berubah!

# Gunakan deep copy jika perlu
import copy
list2 = copy.deepcopy(list1)

# 4. Menggunakan list untuk membership test yang sering
visited = []  # ❌ O(n) tiap 'in'
for node in nodes:
    if node in visited:  # lambat untuk list besar
        continue

# Gunakan set
visited = set()  # ✅ O(1)
for node in nodes:
    if node in visited:
        continue
```
