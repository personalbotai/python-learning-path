# Exceptions: try, except, else, finally
**ID**: `M06-L01`  
**Type**: lesson  
**Duration**: 30-60 menit  
**Tags**: python, tutorial

## Tujuan Pembelajaran
Catch and handle exceptions, understand the exception hierarchy, use else and finally blocks.

## Materi

### Konsep Dasar
Exceptions adalah mekanisme Python untuk menangani errors dan situasi tidak terduga. Gunakan `try-except` untuk menangkap dan menangani exceptions.

**Basic Structure**:
```python
try:
    # kode yang mungkin raise exception
    result = 10 / 0
except ZeroDivisionError:
    # handle exception tertentu
    print("Cannot divide by zero")
except (TypeError, ValueError) as e:
    # handle multiple exceptions
    print(f"Error: {e}")
except Exception as e:
    # catch所有 exceptions (tapi lebih baik spesifik)
    print(f"Unexpected error: {e}")
else:
    # dieksekusi jika tidak ada exception
    print("Operation successful")
finally:
    # selalu dieksekusi (ada exception atau tidak)
    print("Cleanup")
```

### Exception Hierarchy
BaseException → Exception → (specific exceptions)
- Specific: ZeroDivisionError, ValueError, TypeError, IndexError, KeyError, FileNotFoundError, dll
- `Exception` menangkap semua built-in exceptions (kecuar KeyboardInterrupt, SystemExit yang inherit BaseException langsung)

### else dan finally
- `else`: hanya jika try berhasil tanpa exception
- `finally`: untuk cleanup (close file, release resource), selalu dieksekusi, bahkan jika ada `return` di try/except

### Detail Teknis
- Bisa multiple `except` clauses
- `except Exception as e:` menangkap exception dan mengassign ke variable `e`
- `except:` tanpa specifying exception menangkap semua (termasuk KeyboardInterrupt - tidak disarankan)
- `finally` dieksekusi sebelum function return atau exception propagate
- `else` berguna untuk memisahkan kode yang seharusnya tidak dalam try block
- Bisa raise exception baru setelah menangkap: `raise MyError(...) from original_exception` (exception chaining)

## Latihan

### Soal 1
Buat fungsi `safe_divide(a, b)` yang:
- Mencoba `a / b`
- Jika `b == 0`, tangkap ZeroDivisionError dan kembalikan string `"Error: Division by zero"`
- Jika TypeError (misal a atau b bukan number), kembalikan `"Error: Invalid type"`
- Jika success, kembalikan hasil pembagian
- Gunakan try-except-else (finally tidak wajib)

```python
def latihan_1(a, b):
    # Tulis kode kamu di sini
    pass

if __name__ == "__main__":
    print(latihan_1(10, 2))   # 5.0
    print(latihan_1(10, 0))   # Error: Division by zero
    print(latihan_1(10, "2")) # Error: Invalid type
```

**Expected Output**:
```
5.0
Error: Division by zero
Error: Invalid type
```

### Soal 2
Buat fungsi `read_file_safe(filename)` yang:
- Coba buka file dan read semua konten
- Jika file tidak ada (FileNotFoundError), kembalikan `"Error: File not found"`
- Jika I/O error lain, kembalikan `"Error: I/O error"`
- Jika success, kembalikan isi file sebagai string
- Gunakan try-except-else-finally untuk menampilkan "Attempting to read..." di awal dan "Read complete" di finally (print, tidak affect return)

```python
def latihan_2(filename):
    # Tulis kode kamu di sini
    pass

if __name__ == "__main__":
    print(latihan_2("existing.txt"))  # asumsi file ada
    print(latihan_2("missing.txt"))
```

**Expected Output** (contoh):
```
Attempting to read...
Read complete
<isi file>
Attempting to read...
Read complete
Error: File not found
```
