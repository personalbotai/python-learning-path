## Materi

### Konsep Dasar
**Docstrings**: String literal pertama dalam function/class/module yang berfungsi sebagai dokumentasi. Diakses via `__doc__` atau `help()`.

**Style Populer**:
- **Google Style**:
```python
def func(arg1, arg2):
    """Short description.

    Longer description explaining details.

    Args:
        arg1 (int): description
        arg2 (str): description

    Returns:
        bool: description
    """
```
- **NumPy Style**:
```python
def func(arg1, arg2):
    """
    Short description.

    Longer description.

    Parameters
    ----------
    arg1 : int
        description
    arg2 : str
        description

    Returns
    -------
    bool
        description
    """
```

**Type Hints** (PEP 484): Menambahkan informasi tipe ke parameter dan return value.
```python
def greet(name: str) -> str:
    return "Hello " + name
```
Type hints tidak mengubah runtime behavior (tidak enforced), hanya untuk documentation dan static analysis (mypy, IDE).

### typing Module
- Basic: `int`, `str`, `float`, `bool`, `None`
- Containers: `List[int]`, `Dict[str, int]`, `Tuple[int, str]`, `Set[str]`
- Special: `Any` (bisa apa saja), `Optional[T]` (T atau None), `Union[T1, T2]`, `Callable`
- `TypedDict` untuk dictionary dengan keys tertentu
- `Protocol` untuk structural subtyping (duck typing with types)

### Detail Teknis
- Type hints bisa diabaikan oleh interpreter (dynamic typing tetap berlaku)
- `from typing import List, Dict, Tuple, Optional, Union, Callable, Any`
- Python 3.9+: bisa pakai built-in collection types: `list[int]`, `dict[str, int]`
- `Callable[[Arg1Type, Arg2Type], ReturnType]` untuk fungsi sebagai parameter
- `@overload` decorator untuk fungsi dengan multiple signatures
- Docstring dan type hints saling melengkapi: docstring untuk deskripsi, type hints untuk kontrak tipe
