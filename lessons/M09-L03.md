## Materi

### Konsep Dasar
**Asyncio**: Library untuk concurrent programming menggunakan event loop dan coroutines. Cocok untuk I/O-bound tasks dengan many concurrent connections (e.g., web servers, network clients).

**async/await**:
```python
import asyncio

async def fetch_data():
    print("Start")
    await asyncio.sleep(1)  # simulasi I/O
    print("Done")
    return {"data": 123}

async def main():
    result = await fetch_data()
    print(result)

asyncio.run(main())  # Python 3.7+
```

**Event Loop**: Menjalankan coroutines, tasks, dan callbacks.

**Coroutines**: Functions didefinisikan dengan `async def`. Bisa dipanggil, mengembalikan coroutine object. Harus di-`await`-kan untuk mengeksekusi.

**Tasks**: Coroutines yang dijadwalkan di event loop. `asyncio.create_task(coro)` untuk schedule.

### Concurrency dengan asyncio
`await asyncio.gather(coro1, coro2, ...)` untuk run multiple coroutines concurrently.
```python
async def main():
    results = await asyncio.gather(fetch(1), fetch(2), fetch(3))
```

**asyncio.sleep()** untuk delay non-blocking.
**asyncio.Queue** untuk communication between coroutines.

### Detail Teknis
- `asyncio.run(coro)` adalah entry point (membuat event loop baru, jalankan, close)
- `await` hanya bisa di dalam `async def` function
- Blocking calls (seperti `time.sleep()`, synchronous file I/O, blocking network) harus dihindari; gunakan async versions
- `asyncio.sleep(0)` untuk yield control ke event loop
- `asyncio.current_task()` untuk dapat task saat ini
- `asyncio.wait_for(coro, timeout)` untuk timeout
- `asyncio.shield(coro)` untuk protect coro dari cancellation
- `asyncio.CancelledError` untuk cancellation handling
- Bisa tambah callback dengan `task.add_done_callback(func)`
- `asyncio.run_until_complete()` (Python <3.7) atau manual loop
