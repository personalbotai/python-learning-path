## Materi

### Konsep Dasar
**Pickle**: Module untuk serialization (mengubah objek Python menjadi byte stream) dan deserialization (membalikkan). Bisa menyimpan hampir semua objek Python (termasuk functions, classes, dll) tetapi **tidak aman** untuk data dari sumber tidak tepercaya.

**Serialization (Pickling)**:
```python
import pickle

data = {'a': [1, 2.0, 3+4j], 'b': ('string', u'unicode string'), 'c': None}
with open('data.pkl', 'wb') as f:
    pickle.dump(data, f)                # default protocol tertinggi yang tersedia
    pickle.dump(data, f, protocol=pickle.HIGHEST_PROTOCOL)  # explicit
```

**Deserialization (Unpickling)**:
```python
with open('data.pkl', 'rb') as f:
    data = pickle.load(f)
```

### Pickle Protocols
- Protocol 0: ASCII (oldest, compatible)
- Protocol 1: old binary
- Protocol 2: Python 2.3+
- Protocol 3: Python 3.0+ (default for Python 3)
- Protocol 4: Python 3.4+ (large objects)
- Protocol 5: Python 3.8+ (out-of-band data)
Gunakan `pickle.HIGHEST_PROTOCOL` untuk efisiensi.

### Security Warning
**Jangan pernah pickle data dari sumber tidak tepercaya** (seperti user input, internet). Pickle bisa mengeksekusi arbitrary code saat unpickling. Hanya gunakan untuk data yang Anda sendiri buat dan kontrol.

### Alternative: Shelve
`shelve` module menyimpan dictionary-like objek ke file menggunakan pickle di balik layar.
```python
import shelve
with shelve.open('mydata') as db:
    db['key'] = value
    value = db['key']
```

### Detail Teknis
- Pickle bisa meng-handle recursive objects (circular references)
- Beberapa tipe tidak bisa pickle: lambda functions, nested functions, file handles, sockets, threads, dll (harus custom __reduce__)
- `pickle.dumps()` dan `pickle.loads()` untuk byte strings instead of files
- `pickle.DEFAULT_PROTOCOL` = protocol tertinggi untuk versi Python saat ini
- Untuk compatibility antar versi Python, gunakan protocol 2 atau 3
- `pickle.UnpicklingError` untuk error saat unpickling
