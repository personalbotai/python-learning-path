# Writing Tests with unittest and pytest
**ID**: `M10-L01`  
**Type**: lesson  
**Duration**: 30-60 menit  
**Tags**: python, tutorial

## Tujuan Pembelajaran
Write unit tests, use assertions, fixtures, and parametrize tests with pytest.

## Materi

### Konsep Dasar
**Testing**: Memastikan kode berfungsi sesuai yang diharapkan. Unit testing menguji komponen kecil (functions, classes) secara terisolasi.

**unittest** (built-in):
```python
import unittest

def add(a, b):
    return a + b

class TestAdd(unittest.TestCase):
    def test_add_positive(self):
        self.assertEqual(add(2, 3), 5)

    def test_add_negative(self):
        self.assertEqual(add(-1, -1), -2)

if __name__ == '__main__':
    unittest.main()
```
Jalankan: `python test_file.py` atau `python -m unittest discover`.

**pytest** (external, lebih populer):
```python
# test_file.py
def add(a, b):
    return a + b

def test_add():
    assert add(2, 3) == 5
    assert add(-1, -1) == -2
```
Jalankan: `pytest` (butuh install `pytest`).

**Fixtures** (pytest): Setup dan teardown untuk test.
```python
import pytest

@pytest.fixture
def sample_data():
    return [1, 2, 3]

def test_sum(sample_data):
    assert sum(sample_data) == 6
```

**Parametrize** (pytest): Test dengan multiple inputs.
```python
import pytest

@pytest.mark.parametrize("a,b,expected", [
    (1, 2, 3),
    (-1, 1, 0),
    (0, 0, 0),
])
def test_add_param(a, b, expected):
    assert add(a, b) == expected
```

### Assertions
- unittest: `assertEqual`, `assertNotEqual`, `assertTrue`, `assertFalse`, `assertIs`, `assertIsNone`, `assertIn`, `assertRaises`, dll.
- pytest: plain `assert` (lebih simple), dengan introspection yang bagus saat gagal.
- `pytest.raises()` untuk exception testing:
  ```python
  def test_zero_division():
      with pytest.raises(ZeroDivisionError):
          1 / 0
  ```

### Detail Teknis
- unittest menggunakan OOP (TestClass), pytest lebih fungsional
- pytest secara otomatis menemukan file `test_*.py` atau `*_test.py`
- Fixtures bisa scope: function (default), class, module, session
- `pytest` memiliki plugin ekstensif (coverage, mocking, dll)
- `unittest` bagian dari stdlib (tidak perlu install)
- Test organization: tests/ folder terpisah dari source code
- Coverage: `pytest --cov=my_module tests/` untuk melihat kode yang ditest
