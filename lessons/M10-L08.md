## Tujuan Pembelajaran
Create setup.py/pyproject.toml, build distributions, and publish to PyPI (optional).

## Materi

### Konsep Dasar
**Packaging**: Mengorganisir kode menjadi package yang bisa diinstall via `pip`. Package adalah direktori dengan `__init__.py` dan modul-modul lainnya, plus file metadata.

**pyproject.toml** (modern, direkomendasikan):
```toml
[build-system]
requires = ["setuptools>=61.0", "wheel"]
build-backend = "setuptools.build_meta"

[project]
name = "mypackage"
version = "0.1.0"
description = "Short description"
authors = [{name="Your Name", email="you@example.com"}]
readme = "README.md"
license = {text="MIT"}
classifiers = [
    "Programming Language :: Python :: 3",
    "License :: OSI Approved :: MIT License",
]
dependencies = [
    "requests>=2.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=6.0",
    "black",
]

[project.urls]
Homepage = "https://github.com/username/mypackage"
```

**setup.py** (legacy, masih umum):
```python
from setuptools import setup, find_packages

setup(
    name="mypackage",
    version="0.1.0",
    packages=find_packages(),
    install_requires=["requests>=2.0"],
    python_requires=">=3.8",
    entry_points={
        "console_scripts": [
            "mycmd=mypackage.cli:main",
        ],
    },
)
```

### Building Distributions
```bash
python -m pip install --upgrade build twine
python -m build  # creates dist/*.whl dan dist/*.tar.gz
```
- Wheel (`.whl`): binary distribution (pre-compiled, install cepat)
- Source (`.tar.gz`): source distribution (dikompile saat install)

### Publishing to PyPI
```bash
python -m twine upload dist/*
```
Butuh akun PyPI (test: test.pypi.org). Bisa juga upload ke private repository.

### Package Structure
```
mypackage/
├── pyproject.toml
├── README.md
├── LICENSE
├── src/
│   └── mypackage/
│       ├── __init__.py
│       ├── module1.py
│       └── module2.py
└── tests/
    └── test_module1.py
```
Gunakan `src/` layout untuk menghindari import conflict saat development.

### Entry Points
Console scripts: `entry_points={'console_scripts': ['cmd=module:function']}` memungkinkan install command-line tool.

### Versioning
Gunakan [Semantic Versioning](https://semver.org/): `MAJOR.MINOR.PATCH`.

### Detail Teknis
- `find_packages(where="src")` jika paket di `src/`
- `include_package_data=True` untuk include non-Python files (via MANIFEST.in)
- `package_data={"mypackage": ["data/*.json"]}` untuk data files dalam package
- `exclude_package_data` untuk mengecualikan
- `setup.cfg` bisa digunakan bersama `setup.py` untuk deklaratif config
- `twine check dist/*` untuk validasi sebelum upload
- `python -m build --sdist` dan `--wheel` untuk build tertentu
- Dependency management: `install_requires`, `extras_require` (optional dependencies)
- `setup_requires` untuk build-time dependencies (jarang)

## Latihan

### Soal 1
Buat package sederhana `math_utils` dengan struktur:
```
math_utils/
├── pyproject.toml (atau setup.py)
├── README.md
└── src/
    └── math_utils/
        ├── __init__.py (export add, subtract)
        ├── basic.py (def add(a,b), subtract(a,b))
        └── advanced.py (def multiply(a,b), divide(a,b))
```
Isi `pyproject.toml` dengan metadata minimal (name, version, packages via setuptools). Build package dengan `python -m build`. Cek hasil di `dist/`.

```bash
# commands
cd math_utils
python -m build
ls dist/
```

**Expected Output**: file `math_utils-0.1.0-py3-none-any.whl` dan `.tar.gz`.

### Soal 2
Buat package dengan console script entry point. Misal package `greeter` dengan module `cli.py` berisi:
```python
def main():
    import sys
    name = sys.argv[1] if len(sys.argv)>1 else "World"
    print(f"Hello, {name}!")
```
Di `pyproject.toml` atau `setup.py`, tambah:
```toml
[project.scripts]
greet = "greeter.cli:main"
```
Build dan install secara editable: `pip install -e .`. Lalu jalankan `greet Alice` dari terminal.

```bash
pip install -e .
greet Alice
```

**Expected Output**:
```
Hello, Alice!
```
