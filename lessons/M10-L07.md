## Tujuan Pembelajaran
Profile code with cProfile, timeit, and line_profiler; identify bottlenecks.

## Materi

### Konsep Dasar
**Profiling**: Mengukur performance code (waktu eksekusi, function calls) untuk menemukan bottleneck.

**cProfile**: Built-in deterministic profiler (mengukur function calls dan waktu).
```bash
python -m cProfile -o profile.out my_script.py
```
Atau dalam kode:
```python
import cProfile
import pstats

profiler = cProfile.Profile()
profiler.enable()
# code to profile
profiler.create_stats()
stats = pstats.Stats(profiler)
stats.sort_stats('cumulative').print_stats(10)  # top 10 by cumulative time
```

**timeit**: Untuk mengukur small code snippets.
```python
import timeit

time = timeit.timeit('"-".join(str(n) for n in range(100))', number=10000)
print(time)
```
Atau di command line: `python -m timeit '"-".join(str(n) for n in range(100))'`.

**line_profiler** (external): Profiler per baris kode. Butuh install `line_profiler` dan decorator `@profile`.
```bash
pip install line_profiler
kernprof -l -v my_script.py
```

### Interpreting Results
- **cProfile**: kolom `ncalls` (number of calls), `tottime` (time dalam fungsi itu sendiri), `percall` (tottime/ncalls), `cumtime` (total time termasuk fungsi yang dipanggil), `percall` (cumtime/ncalls)
- Fokus pada fungsi dengan `cumtime` tinggi atau `tottime` tinggi (bisa jadi recursive)
- `line_profiler` menunjukkan waktu per baris, membantu menemukan baris spesifik yang lambat

### Optimization Strategies
- Choose right data structures (list vs set untuk membership)
- Avoid unnecessary computations (cache results, memoization)
- Use built-in functions (lebih cepat)
- Algorithmic improvements (O(n^2) → O(n log n))
- Local variables (lebih cepat daripada global)
- Loop optimizations (avoid work inside loop jika bisa)
- Use `join()`Instead of repeated string concatenation

### Detail Teknis
- `pstats.Stats` bisa save to file, sort by different columns (`'time'`, `'calls'`, `'cumulative'`)
- `cProfile` memiliki overhead; untuk production gunakan sampling profiler (seperti `yappi` atau `py-spy`)
- `timeit` otomatis disable garbage collector; bisa di-enable dengan `setup='gc.enable()'`
- `line_profiler` memerlukan decorator `@profile` pada fungsi yang ingin di-profile; jalankan dengan `kernprof`
- Profiling memory: `memory_profiler` module (line-by-line memory usage)
- `snakeviz` untuk visualize cProfile output (graphical)

## Latihan

### Soal 1
Buat dua fungsi: `sum_squares(n)` yang menghitung jumlah kuadrat dari 1 sampai n dengan loop, dan `sum_squares_formula(n)` yang menggunakan rumus matemática `n*(n+1)*(2n+1)/6`. Profile kedua fungsi untuk `n=1000000` menggunakan `timeit`. Bandingkan waktu.

```python
import timeit

def sum_squares(n):
    total = 0
    for i in range(1, n+1):
        total += i * i
    return total

def sum_squares_formula(n):
    return n * (n+1) * (2*n+1) // 6

def latihan_1():
    n = 1000000
    time_loop = timeit.timeit(lambda: sum_squares(n), number=10)
    time_formula = timeit.timeit(lambda: sum_squares_formula(n), number=10)
    print(f"Loop: {time_loop:.4f}s")
    print(f"Formula: {time_formula:.4f}s")

if __name__ == "__main__":
    latihan_1()
```

**Expected Output** (contoh):
```
Loop: 0.5432s
Formula: 0.0001s
```

### Soal 2
Gunakan `cProfile` untuk menganalisis fungsi yang melakukan banyak recursive calls (misal: Fibonacci naive). Tampilkan top 5 fungsi berdasarkan `cumulative` time.identifikasi bottleneck.

```python
import cProfile
import pstats

def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

def latihan_2():
    profiler = cProfile.Profile()
    profiler.enable()
    result = fib(30)
    profiler.create_stats()
    stats = pstats.Stats(profiler)
    stats.sort_stats('cumulative').print_stats(5)
    print(f"Result: {result}")

if __name__ == "__main__":
    latihan_2()
```

**Expected Output** (contoh):
```
   ncalls  tottime  percall  cumtime  percall filename:lineno(function)
  1346263    0.200    0.000    0.200    0.000 <string>:1(fib)
        1    0.000    0.000    0.200    0.200 <string>:1(<module>)
...
Result: 832040
```
