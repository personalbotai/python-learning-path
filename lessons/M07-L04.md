## Tujuan Pembelajaran
Manipulate file paths with os.path and the modern pathlib API.

## Materi

### Konsep Dasar
**os.path**: Module untuk operasi path berbasis string.
```python
import os.path

path = '/home/user/file.txt'
os.path.join('dir', 'subdir', 'file.txt')
os.path.split(path)          # ('/home/user', 'file.txt')
os.path.dirname(path)       # '/home/user'
os.path.basename(path)      # 'file.txt'
os.path.splitext(path)      # ('/home/user/file', '.txt')
os.path.exists(path)
os.path.isfile(path)
os.path.isdir(path)
os.path.getsize(path)
```

**pathlib** (Python 3.4+): OOP approach.
```python
from pathlib import Path

p = Path('dir') / 'subdir' / 'file.txt'   # operator overloading
p.exists()
p.is_file()
p.is_dir()
p.name          # 'file.txt'
p.stem          # 'file'
p.suffix        # '.txt'
p.parent        # Path('dir/subdir')
p.read_text()
p.write_text('content')
p.read_bytes()
p.write_bytes(b'content')
p.mkdir(parents=True, exist_ok=True)
p.rmdir()
p.unlink()      # delete file
p.glob('*.py')  # pattern matching
p.iterdir()
```

### Detail Teknis
- `os.path` selalu pakai string; `pathlib` menggunakan objek Path (lebih readable)
- `Path` bisa di-overload dengan `/` operator: `p = Path('a') / 'b'` → `a/b`
- `Path` methods return Path objects (chainable)
- `Path.home()`, `Path.cwd()` untuk path umum
- `os.path` masih valid dan sering dipakai di code legacy; `pathlib` direkomendasikan untuk code baru
- `os.path.abspath()` untuk path absolut
- `os.path.normpath()` untuk normalisasi (resolve `.` dan `..`)
- `os.path.samefile()` cek apakah dua path menuju file yang sama
- `os.path.realpath()` untuk resolve symbolic links
