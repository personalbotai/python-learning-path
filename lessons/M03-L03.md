## Tujuan Pembelajaran
Understand tuple immutability, packing/unpacking, and when to use tuples vs lists.

## Materi

### Konsep Dasar
Tuple adalah immutable sequence yang一旦 dibuat, elemennya tidak bisa diubah (tidak bisa tambah/hapus/ubah).

**Creating Tuples**:
- `t = (1, 2, 3)`
- `t = tuple(iterable)`
- Single element tuple: `(1,)` (perlu koma)
- Empty tuple: `()`

**Tuple Packing & Unpacking**:
- Packing: `t = 1, 2, 3` (tanpa parentheses)
- Unpacking: `a, b, c = t`
- Extended unpacking: `a, *rest, b = t`
- Swap dengan unpacking: `a, b = b, a`

### Immutability
- Tidak bisa: assign elemen, append, extend, remove, pop
- Bisa: contains, indexing, slicing (hasilnya tuple baru), concatenation (membuat tuple baru)
- Keamanan: immutable → bisa digunakan sebagai key dalam dictionary
- Performance: lebih cepat dan hemat memory dibanding list

### When to Use Tuples vs Lists
- **Tuple**: data yang seharusnya tidak berubah, heterogeneous (tipe berbeda), sebagai record/struct, dictionary keys
- **List**: data yang mungkin berubah, homogeneous (tipe sama), mutable collection

### Detail Teknis
- Tuple methods: `count()`, `index()` (hanya dua karena immutable)
- Slicing tuple menghasilkan tuple baru
- Concatenation `+` membuat tuple baru
- Tuple bisa berisi mutable objects (list di dalam tuple bisa diubah, tapi tuple structure tetap)
- Named tuple (`collections.namedtuple`) untuk readability
