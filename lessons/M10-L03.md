# Debugging with pdb and IDE Debuggers
**ID**: `M10-L03`

## Tujuan Pembelajaran
Set breakpoints, step through code, inspect variables, use post-mortem debugging.

## Materi

### Konsep Dasar
**Debugging**: Proses menemukan dan memperbaiki bugs. Python memiliki debugger bawaan `pdb` (Python Debugger) dan semua IDE modern memiliki debugger GUI.

**pdb Commands**:
- `l` (list): tampilkan kode di sekitar posisi saat ini
- `n` (next): eksekusi baris berikutnya (step over)
- `s` (step): step into function call
- `c` (continue): lanjutkan sampai breakpoint berikutnya atau selesai
- `r` (return): continue sampai return dari current function
- `p expr` (print): evaluate expression dan print
- `pp expr`: pretty-print
- `b [file:]lineno` atau `b func`: set breakpoint
- `cl` atau `clear`: hapus breakpoint
- `w` (where): print stack trace
- `u` (up) / `d` (down): navigate stack frames
- `q` (quit): keluar dari debugger

**Using pdb**:
```python
import pdb

def buggy_function(x):
    pdb.set_trace()  # breakpoint
    result = x / (x - 5)
    return result

buggy_function(10)
```
Jalankan script, akan berhenti di `set_trace()`. Atau: `python -m pdb script.py` untuk debug dari awal.

**Post-mortem Debugging**: Setelah exception, gunakan `import pdb; pdb.pm()` untuk masuk ke debugger di posisi where exception terjadi.

**IDE Debuggers**:
- VS Code, PyCharm, dll: breakpoint via klik, watch variables, step over/into, continue, evaluate expression, conditional breakpoints.
- Lebih user-friendly, graphical.

### Detail Teknis
- `breakpoint()` (Python 3.7+) adalah function yang memanggil debugger sesuai `sys.breakpointhook`. Default `pdb.set_trace()`. Bisa ubah dengan env `PYTHONBREAKPOINT`.
- `pdb.Pdb()` bisa subclass untuk custom commands.
- `pdb.run(statement)` atau `pdb.runcall(function, *args, **kwargs)` untuk programmatic debugging.
- `pdb.post_mortem(traceback)` untuk post-mortem.
- Conditional breakpoints: `b 42, x > 5` (break di line 42 jika x>5)
- Ignore count: `b func, ignore:10` (skip 10 kali)
- `display expr`: auto-print expr setiap stop
- `undisplay` untuk hapus display
- `pdb` bisa remote debugging via `pdb.runcall` atau `pdb.set_trace()`

## Latihan

### Soal 1
Gunakan `pdb` untuk men-debug fungsi berikut yang memiliki bug (off-by-one). Set breakpoint, step through, periksa variabel, temukan bug, dan perbaiki.

```python
def sum_range(n):
    total = 0
    for i in range(1, n):
        total += i
    return total

if __name__ == "__main__":
    print(sum_range(5))  # seharusnya 1+2+3+4 = 10, tapi output?
```

**Tugas**: Tambah `pdb.set_trace()`, jalankan, cek nilai `i` dan `total` tiap iterasi, temukan bahwa `range(1,n)` seharusnya `range(1, n+1)` atau `range(n+1)`. Perbaiki.

**Expected Output** (setelah perbaikan):
```
10
```

### Soal 2
Buat fungsi rekursif `factorial(n)` yang memiliki bug (tidak handle base case 0). Gunakan post-mortem debugging: panggil fungsi dengan nilai yang akan cause exception, lalu import pdb dan `pdb.pm()`.

```python
def factorial(n):
    return n * factorial(n-1)  # missing base case

if __name__ == "__main__":
    try:
        print(factorial(5))
    except RecursionError:
        import pdb
        pdb.pm()  # debug di posisi exception
```

**Tugas**: Setelah masuk ke pdb, lihat stack trace (`w`), cek nilai `n` di setiap frame, temukan bahwa base case `if n == 0: return 1` tidak ada. Tambahkan base case dan verifikasi.

**Expected Output** (setelah perbaikan):
```
120
```
