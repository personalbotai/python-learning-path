# Metaclasses and Class Creation
**ID**: `M08-L06`  
**Type**: lesson  
**Duration**: 30-60 menit  
**Tags**: python, tutorial

## Tujuan Pembelajaran
Understand type(), create custom metaclasses, and know when to use them (rarely).

## Materi

### Konsep Dasar
**Metaclass**: Class dari class. Class adalah instance dari metaclass. Default metaclass adalah `type`.

```python
class MyClass:
    pass
# Di balik layar: MyClass = type('MyClass', (object,), {})
```

**type()** bisa digunakan untuk membuat class secara dinamis:
```python
MyClass = type('MyClass', (base1, base2), {'attr': value, 'method': lambda self: ...})
```

**Custom Metaclass**: Inherit dari `type` dan override `__new__` atau `__init__`.
```python
class Meta(type):
    def __new__(mcs, name, bases, attrs):
        # mcs: metaclass instance (Meta)
        # name: nama class yang akan dibuat
        # bases: tuple base classes
        # attrs: dict attributes (methods, class variables)
        # modify attrs jika perlu
        cls = super().__new__(mcs, name, bases, attrs)
        return cls

class MyClass(metaclass=Meta):
    pass
```

### Use Cases (Rare)
- Framework development (ORM, API schema, serialization)
- Automatic registration of subclasses
- Enforcing coding conventions (misal: semua methods harus punya docstring)
- Singleton pattern (dengan override `__call__`)
- API versioning

### __init__ vs __new__ dalam Metaclass
- `__new__`: membuat class object (langsung memodifikasi dict sebelum class dibuat)
- `__init__`: menginisialisasi class object setelah dibuat (lebih umum digunakan)

### Detail Teknis
- `__prepare__` bisa didefinisikan di metaclass untuk mengembalikan custom mapping (misal OrderedDict) untuk menyimpan atribut class dalam urutan definisi (Python 3.6+ sudah preserve order by default)
- Metaclass bisa mengubah inheritance (`bases`) atau menambahkan attributes ke `attrs`
- Class dengan metaclass tidak bisa di-instance jika metaclass `__call__` tidak mengembalikan instance (misal untuk abstract classes)
- `type` adalah kelas itu sendiri (type of type is type)
- Banyak pattern yang bisa dilakukan dengan class decorators; metaclasses lebih powerful tapi lebih kompleks

## Latihan

### Soal 1
Buat metaclass `AddRepr` yang otomatis menambahkan `__repr__` method ke class yang menggunakannya. `__repr__` harus mengembalikan `"ClassName(attr1=value1, attr2=value2, ...)"` untuk semua instance attributes (dari `self.__dict__`). Jangan override jika class sudah punya `__repr__`.

```python
class AddRepr(type):
    def __new__(mcs, name, bases, attrs):
        # Tulis kode kamu di sini
        pass

class Person(metaclass=AddRepr):
    def __init__(self, name, age):
        self.name = name
        self.age = age

if __name__ == "__main__":
    p = Person("Alice", 25)
    print(repr(p))  # Person(name='Alice', age=25)
```

**Expected Output**:
```
Person(name='Alice', age=25)
```

### Soal 2
Buat metaclass `EnforceTypes` yang memeriksa semua type hints dalam methods (gunakan `__annotations__`). Saat `__new__`, jika ada type hints, store them. Lalu, override `__call__` (metaclass `__call__` adalah saat instance dibuat) untuk validate arguments yang diberikan ke `__init__` terhadap type hints. Jika type tidak sesuai, raise TypeError. Hanya check attributes yang diannotate di `__init__` (jangan methods lain).

```python
class EnforceTypes(type):
    def __new__(mcs, name, bases, attrs):
        # collect hints from __init__ if exists
        # store di cls.__type_hints__
        pass

    def __call__(cls, *args, **kwargs):
        # bind args ke __init__ parameters
        # validate type sesuai hints
        # jika ok, panggil super().__call__()
        pass

class Person(metaclass=EnforceTypes):
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

if __name__ == "__main__":
    p = Person("Bob", 30)   # OK
    p2 = Person(123, 30)    # TypeError: name harus str
```

**Expected Output**:
```
TypeError: Argument 'name' must be type <class 'str'>, got <class 'int'>
```
